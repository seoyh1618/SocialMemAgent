Guide de qualité du code basé sur les principes fondamentaux de Toss Frontend. Fournit des commentaires en utilisant quatre critères (lisibilité, prévisibilité, cohésion et couplage) lors des révisions de code, de l'écriture de nouveaux codes et de la refactorisation. Activé pour la qualité du code frontend, le code propre et les demandes d'amélioration du code.

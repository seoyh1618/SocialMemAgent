Générez/maintenez des tests BDD/ezSpec à partir du fichier accept.yaml dans le catalogue de spécifications. En utilisant une syntaxe de type Gherkin, l'IA génère automatiquement des définitions d'étapes (les développeurs n'ont pas besoin de les écrire à la main), de sorte que la spécification du test d'acceptation est une spécification exécutable.

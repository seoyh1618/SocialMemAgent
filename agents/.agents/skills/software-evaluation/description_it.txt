Valuta la qualità del codice in base a cinque pilastri (architettura, affidabilità, osservabilità, sicurezza, DX) e produci una scorecard 1-10 con una roadmap di miglioramento strategico. Da utilizzare quando si desidera una revisione completa della qualità di una directory o di un modulo. Si attiva su richieste come "valutazione della qualità del codice", "revisione dell'architettura", "punteggio del codice", "audit del codice", "/software.evaluation" o quando viene richiesta una valutazione della qualità di una directory dal punto di vista di un CTO o di un architetto.

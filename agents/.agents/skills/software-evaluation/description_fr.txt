Évaluez la qualité du code sur cinq piliers (architecture, fiabilité, observabilité, sécurité, DX) et produisez une carte de pointage de 1 à 10 avec une feuille de route d'amélioration stratégique. À utiliser lorsque vous souhaitez un examen complet de la qualité d’un répertoire ou d’un module. Se déclenche sur des requêtes telles que « évaluer la qualité du code », « examiner l'architecture », « évaluer mon code », « audit de code », « /software.evaluation » ou lorsqu'on lui demande une évaluation de la qualité d'un répertoire du point de vue d'un CTO ou d'un architecte.

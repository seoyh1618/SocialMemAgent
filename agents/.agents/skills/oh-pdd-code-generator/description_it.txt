Genera automaticamente un framework di codice HarmonyOS da documenti di progettazione, incluse definizioni di interfaccia IDL, struttura di directory, file di intestazione, modelli di file di origine e configurazioni di build. Prima della generazione, analizza il codice OpenHarmony esistente per apprendere lo stile del codice corrente e le norme dell'architettura. Applicabile alle richieste dell'utente: (1) generare framework di codice, (2) creare interfacce IDL, (3) generare configurazioni BUILD.gn, (4) creare codice di servizio, (5) inizializzare la struttura del progetto, (6) seguire lo stile del codice esistente. Parole chiave: generazione di codice, IDL, BUILD.gn, bundle.json, profilo SA, stile di codice OpenHarmony, generazione di codice, definizione di interfaccia, configurazione di build, stile di codice OH

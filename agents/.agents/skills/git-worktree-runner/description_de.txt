Anleitung zur Verwendung von gtr (git-worktree-runner), einem CLI-Tool zur Verwaltung von Git-Worktrees mit Editor- und AI-Tool-Integration. Verwenden Sie diese Fähigkeit, wenn der Benutzer Folgendes fragt: (1) Erstellen, Verwalten oder Entfernen von Git-Arbeitsbäumen, (2) Einrichten von GTR / Git-GTR für ein Projekt, (3) Konfigurieren von Editoren (Cursor, VS Code, Zed) oder KI-Tools (Claude Code, Aider, Codex, Gemini) mit Arbeitsbäumen, (4) Ausführen paralleler Entwicklungsworkflows über mehrere Zweige hinweg, (5) Kopieren von Konfigurations-/Env-Dateien in neue Arbeitsbäume, (6) Einstellung Up-Hooks für die Erstellung/Entfernung von Worktree, (7) Fehlerbehebung bei Git-Worktree-Problemen, (8) jegliche Erwähnung von „gtr“, „git gtr“, „git worktree runner“ oder „worktree“ im Zusammenhang mit paralleler Entwicklung. Löst auf Schlüsselwörtern aus: gtr, git-gtr, Arbeitsbaum, parallele Zweige, Workflow mit mehreren Zweigen.

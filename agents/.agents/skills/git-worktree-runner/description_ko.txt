에디터와 AI 도구가 통합된 git 작업트리 관리용 CLI 도구인 gtr(git-worktree-runner) 사용 가이드입니다. 사용자가 (1) git 작업 트리 생성, 관리 또는 제거, (2) 프로젝트에 대한 gtr/git-gtr 설정, (3) 작업 트리를 사용하여 편집기(Cursor, VS Code, Zed) 또는 AI 도구(Claude Code, Aider, Codex, Gemini) 구성, (4) 여러 분기에 걸쳐 병렬 개발 워크플로 실행, (5) config/env 파일을 새 작업 트리에 복사, (6) 후크 설정에 관해 질문할 때 이 기술을 사용합니다. 작업 트리 생성/제거의 경우, (7) git 작업 트리 문제 해결, (8) 병렬 개발의 맥락에서 "gtr", "git gtr", "git worktreerunner" 또는 "worktree"에 대한 언급. 키워드에 따라 트리거: gtr, git-gtr, worktree, 병렬 분기, 다중 분기 워크플로.

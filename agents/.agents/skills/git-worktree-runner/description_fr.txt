Guide d'utilisation de gtr (git-worktree-runner), un outil CLI pour gérer les arbres de travail git avec l'intégration d'un éditeur et d'un outil d'IA. Utilisez cette compétence lorsque l'utilisateur pose des questions sur : (1) la création, la gestion ou la suppression d'arbres de travail git, (2) la configuration de gtr / git-gtr pour un projet, (3) la configuration d'éditeurs (Cursor, VS Code, Zed) ou d'outils d'IA (Claude Code, Aider, Codex, Gemini) avec des arbres de travail, (4) l'exécution de flux de travail de développement parallèles sur plusieurs branches, (5) la copie de fichiers de configuration/env dans de nouveaux arbres de travail, (6) la configuration des crochets pour la création/suppression d'arbre de travail, (7) le dépannage des problèmes de git worktree, (8) toute mention de "gtr", "git gtr", "git worktree runner" ou "worktree" dans le contexte d'un développement parallèle. Déclencheurs sur mots-clés : gtr, git-gtr, worktree, branches parallèles, workflow multi-branches.

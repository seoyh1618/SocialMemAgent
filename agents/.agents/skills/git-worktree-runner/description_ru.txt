Руководство по использованию gtr (git-worktree-runner), инструмента CLI для управления рабочими деревьями git с интеграцией редактора и инструментов искусственного интеллекта. Используйте этот навык, когда пользователь спрашивает о: (1) создании, управлении или удалении рабочих деревьев git, (2) настройке gtr / git-gtr для проекта, (3) настройке редакторов (Cursor, VS Code, Zed) или инструментов искусственного интеллекта (Claude Code, Aider, Codex, Gemini) с рабочими деревьями, (4) запуске параллельных рабочих процессов разработки в нескольких ветках, (5) копировании файлов config/env в новые рабочие деревья, (6) настройка перехватчиков для создания/удаления рабочего дерева, (7) устранение проблем с рабочим деревом git, (8) любое упоминание «gtr», «git gtr», «git worktree runner» или «worktree» в контексте параллельной разработки. Триггеры по ключевым словам: gtr, git-gtr, рабочее дерево, параллельные ветки, многоветвевой рабочий процесс.

Fastify の 5 つのベスト プラクティス、API リファレンス、ルート、プラグイン、フック、検証、エラー処理、TypeScript のパターン。次の場合に使用します: (1) 新しい Fastify ルート、プラグイン、またはフックを作成する場合、(2) Fastify API 署名またはオプションを検索する場合、(3) Fastify の問題 (ライフサイクル、カプセル化、検証、プラグインのタイムアウト) をデバッグする場合、(4) アンチパターンについて Fastify コードをレビューする場合。トリガー: 'ルートの追加'、'プラグインの作成'、'Fastify フック'、'検証スキーマ'、'Fastify エラー'、'setErrorHandler'、'fastify-plugin'。

Verwenden Sie @aptx/token-store, um die Token-Persistenz zu definieren oder zu implementieren. Für: Implementierung der TokenStore-Schnittstelle, Unterstützung von sync/async-APIs (getToken/setToken/clearToken), optionale Metadatenmethoden, Implementierung verschiedener Speicher-Backends (Cookie, localStorage, Miniprogramm, Speicher) und Verwendung mit @aptx/api-plugin-auth. Wird ausgelöst, wenn Code einen benutzerdefinierten TokenStore implementieren muss.

Utilice @aptx/token-store para definir o implementar la persistencia del token. Propósito: implementar la interfaz TokenStore, admitir API de sincronización/asíncrono (getToken/setToken/clearToken), métodos de metadatos opcionales, implementar diferentes backends de almacenamiento (cookie, localStorage, miniprograma, memoria) y trabajar con @aptx/api-plugin-auth. Se activa cuando el código necesita implementar un TokenStore personalizado.

Utilizzare @aptx/token-store per definire o implementare la persistenza del token. Utilizzato per: implementare l'interfaccia TokenStore, supportare API di sincronizzazione/asincrona (getToken/setToken/clearToken), metodi di metadati opzionali, implementare diversi backend di archiviazione (cookie, localStorage, mini-programma, memoria) e utilizzato con @aptx/api-plugin-auth. Attivato quando il codice deve implementare un TokenStore personalizzato.

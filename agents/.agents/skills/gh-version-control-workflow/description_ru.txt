Запустите дисциплинированный рабочий процесс GitHub с помощью git + gh, используя задачи, ветки, производные от задач, рабочие деревья и PR. Используйте, когда вам нужна параллельная работа над функциями, четкие границы проверки и PR, связанные с проблемами, вместо прямых коммитов в ветки интеграции.

Exécutez un workflow GitHub discipliné avec git + gh en utilisant des tickets, des branches dérivées de tickets, des arbres de travail et des PR. À utiliser lorsque vous avez besoin d'un travail simultané sur les fonctionnalités, de limites de révision claires et de PR liés à des problèmes au lieu de engagements directs dans les branches d'intégration.

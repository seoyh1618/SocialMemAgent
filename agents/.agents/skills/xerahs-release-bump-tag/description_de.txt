Orchestrieren Sie den XerahS-Release-Flow in strenger Reihenfolge: zuerst Wartungsarbeiten, dann Update-Änderungsprotokoll (optional, wenn kein CHANGELOG vorhanden ist), überprüfen Sie Build, Bump/Commit/Push/Tag während der Synchronisierung der Chocolatey-Versionsmetadaten, überwachen Sie GitHub-Aktionen alle 2 Minuten, stellen Sie den Standardinhalt der Versionshinweise sicher und legen Sie dann optional eine Vorabversion fest. Überprüfen Sie bei Fehlern die Protokolle, beheben Sie die Grundursache und versuchen Sie es mit der nächsten Patch-Version erneut.

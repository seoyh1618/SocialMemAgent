قم بتنظيم تدفق إصدار XerahS بترتيب صارم: أعمال الصيانة أولاً، وتحديث سجل التغيير ثانيًا (اختياري إذا لم يكن هناك سجل تغيير)، والتحقق من البناء، والارتداد/الالتزام/الدفع/العلامة أثناء مزامنة البيانات التعريفية لإصدار Chocolatey، ومراقبة إجراءات GitHub كل دقيقتين، والتأكد من محتوى ملاحظات الإصدار القياسي، ثم تعيين الإصدار المسبق بشكل اختياري. في حالة الفشل، افحص السجلات وأصلح السبب الجذري وأعد المحاولة باستخدام إصدار التصحيح التالي.

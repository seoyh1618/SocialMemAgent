ドキュメント駆動開発 (文档驱动开発行)。次の場合に使用します: (1) /dd help — ワークフローを説明し、プロジェクトの状態に基づいて次のステップを提案する、(2) /dd init — プロジェクト内のドキュメント主導の構造を初期化する、(3) /dd spec [requirement] — 開発前に意図を調整するために一時的なスペックを作成する、(4) /dd dev [spec-path|doc-path] — spec または doc に従ってコードを実装する (スペックベースの開発の計画モードに自動的に入る)、(5) /dd sync [--check|--scope] — 生きているドキュメント、アーカイブ仕様、および監査の一貫性を更新します。次の言及によってトリガーされます: ドキュメント駆動、設計ドキュメント、ドキュメント駆動、仕様駆動、文档驱动、设计文档、规格、/dd。

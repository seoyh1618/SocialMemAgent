Développement piloté par documents (文档驱动开发). À utiliser lorsque : (1) /dd help — explique le flux de travail et suggère l'étape suivante en fonction de l'état du projet, (2) /dd init — initialise la structure basée sur un document dans un projet, (3) /dd spec [exigence] — crée une spécification temporaire pour aligner l'intention avant le développement, (4) /dd dev [spec-path|doc-path] — implémente le code suivant la spécification ou le doc (entre automatiquement en mode plan pour le développement basé sur les spécifications), (5) /dd sync [--check|--scope] — met à jour live documents, spécifications d'archives et cohérence des audits. Déclencheurs sur les mentions de : piloté par document, document de conception, piloté par doc, piloté par spécifications, 文档驱动, 设计文档, 规格, /dd.

Surveillez une demande d'extraction GitHub après la création en interrogeant en permanence les vérifications CI/exécutions de workflow, les nouveaux commentaires de révision et l'état de fusion jusqu'à ce que le PR soit prêt à fusionner (ou fusionné/fermé). Diagnostiquez les échecs, réessayez les échecs probables jusqu'à 3 fois, corrigez/poussez automatiquement les problèmes liés aux branches le cas échéant et arrêtez-vous uniquement lorsque l'aide de l'utilisateur est requise (par exemple, des problèmes d'infrastructure CI, des tentatives instables épuisées ou des situations ambiguës/bloquantes). À utiliser lorsque l'utilisateur demande à Codex de surveiller un PR, de surveiller le CI, de gérer les commentaires de révision ou de garder un œil sur les échecs et les commentaires sur un PR ouvert.

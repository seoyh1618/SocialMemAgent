Cuide una solicitud de extracción de GitHub después de su creación sondeando continuamente las verificaciones de CI/ejecuciones de flujo de trabajo, nuevos comentarios de revisión y el estado de fusionabilidad hasta que el PR esté listo para fusionarse (o fusionarse/cerrarse). Diagnostique fallas, vuelva a intentar fallas probables hasta 3 veces, corrija automáticamente/envíe problemas relacionados con ramas cuando sea apropiado y deténgase solo cuando se requiera ayuda del usuario (por ejemplo, problemas de infraestructura de CI, reintentos fallidos agotados o situaciones ambiguas/de bloqueo). Úselo cuando el usuario solicite a Codex que supervise un RP, observe CI, maneje comentarios de revisión o vigile las fallas y los comentarios sobre un RP abierto.

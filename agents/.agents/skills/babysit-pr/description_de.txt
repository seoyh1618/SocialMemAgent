Betreuen Sie einen GitHub-Pull-Request nach der Erstellung, indem Sie kontinuierlich CI-Prüfungen/Workflow-Läufe, neue Überprüfungskommentare und den Zusammenführungsstatus abfragen, bis der PR zum Zusammenführen (oder zusammenführen/schließen) bereit ist. Diagnostizieren Sie Fehler, wiederholen Sie wahrscheinliche fehlerhafte Fehler bis zu dreimal, beheben Sie verzweigungsbezogene Probleme bei Bedarf automatisch/pushen Sie sie und stoppen Sie nur, wenn Benutzerhilfe erforderlich ist (z. B. Probleme mit der CI-Infrastruktur, erschöpfte fehlerhafte Wiederholungsversuche oder unklare/blockierende Situationen). Verwenden Sie diese Option, wenn der Benutzer Codex auffordert, einen PR zu überwachen, CI zu überwachen, Überprüfungskommentare zu bearbeiten oder Fehler und Feedback zu einem offenen PR im Auge zu behalten.

PR のマージの準備ができる (またはマージ/クローズされる) まで、CI チェック/ワークフローの実行、新しいレビュー コメント、マージ可能性の状態を継続的にポーリングすることで、GitHub プル リクエストの作成後にベビーシッターを行います。障害を診断し、不安定な失敗の可能性がある場合は最大 3 回再試行し、必要に応じてブランチ関連の問題を自動修正/プッシュし、ユーザーのヘルプが必要な場合にのみ停止します (CI インフラストラクチャの問題、不安定な再試行の限界、曖昧な/ブロックされた状況など)。ユーザーが Codex に PR の監視、CI の監視、レビュー コメントの処理、またはオープン PR の失敗やフィードバックの監視を依頼する場合に使用します。

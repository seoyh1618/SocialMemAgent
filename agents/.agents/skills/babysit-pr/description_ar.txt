قم برعاية طلب سحب GitHub بعد الإنشاء من خلال الاستطلاع المستمر لعمليات فحص CI/عمليات سير العمل، وتعليقات المراجعة الجديدة، وحالة قابلية الدمج حتى يصبح العلاقات العامة جاهزة للدمج (أو الدمج/الإغلاق). تشخيص حالات الفشل، وإعادة محاولة حالات الفشل غير المستقرة المحتملة حتى 3 مرات، والإصلاح التلقائي/دفع المشكلات المتعلقة بالفرع عندما يكون ذلك مناسبًا، والتوقف فقط عند الحاجة إلى مساعدة المستخدم (على سبيل المثال مشكلات البنية التحتية لـ CI، أو إعادة المحاولة غير المستقرة المستنفدة، أو المواقف الغامضة/الحظر). يُستخدم عندما يطلب المستخدم من Codex مراقبة العلاقات العامة، أو مشاهدة CI، أو التعامل مع تعليقات المراجعة، أو مراقبة حالات الفشل والتعليقات على العلاقات العامة المفتوحة.

Assisti a una richiesta pull di GitHub dopo la creazione eseguendo il polling continuo di controlli CI/esecuzioni del flusso di lavoro, nuovi commenti di revisione e stato di unificabilità finché il PR non è pronto per l'unione (o unito/chiuso). Diagnostica gli errori, riprova i probabili errori instabili fino a 3 volte, corregge automaticamente/invia i problemi relativi ai rami quando appropriato e interrompi solo quando è richiesto l'aiuto dell'utente (ad esempio problemi dell'infrastruttura CI, tentativi instabili esauriti o situazioni ambigue/bloccanti). Da utilizzare quando l'utente chiede al Codex di monitorare un PR, guardare CI, gestire i commenti di revisione o tenere d'occhio i fallimenti e il feedback su un PR aperto.

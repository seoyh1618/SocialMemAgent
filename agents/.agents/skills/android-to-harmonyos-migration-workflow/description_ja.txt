マルチエージェントのコラボレーションによる、Android から HarmonyOS への反復的なコード移行ワークフロー。ユーザーが Android プロジェクトを HarmonyOS に移行する必要があるときにトリガーされ、次の機能が提供されます。 1) Analyzer エージェントがコード構造をスキャン 2) Planner エージェントが移行計画を作成 3) Translator エージェントがコード変換を実行 4) Validator エージェントが ohos-app-build-debug スキルを使用してコンパイル検証、アプリのパッケージ化、およびデバイス上の最初の画面検証を行う 5) Tester エージェントが機能を検証する (各モジュールの移行後に実行) 6) すべてのモジュールの実行後移行、Feature Comparator Agent が欠落している機能をチェックします。 7) UI Comparator Agent が UI の一貫性を検証します。 8) 未完成の部分を Planner Agent にフィードバックして、さらなる反復を行います。コード分​​析スクリプト、API マッピング テーブル、コンポーネント変換パターン、自動検証ツールが含まれます。トリガーキーワード: Android から HarmonyOS への移行、Android の移行、HarmonyOS の移行、Java/Kotlin から ArkTS へ、アクティビティからページ、機能比較、UI 比較。

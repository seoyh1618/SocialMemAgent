Flujo de trabajo de migración de código colaborativo de múltiples agentes de Android a HarmonyOS (iterativo). Se activa cuando un usuario necesita migrar un proyecto de Android a HarmonyOS y proporciona: 1) el Agente analizador escanea la estructura del código 2) El Agente planificador redacta el plan de migración 3) El Agente traductor realiza la conversión del código 4) **El Agente validador** utiliza la habilidad **ohos-app-build-debug** para la verificación de compilación, el empaquetado de aplicaciones y la validación en la primera pantalla en el dispositivo 5) El Agente probador verifica la funcionalidad (se ejecuta después de cada migración de módulo) 6) Después de migrar todos los módulos, el Agente comparador de características comprueba las funciones faltantes 7) El agente comparador de UI valida la coherencia de la interfaz de usuario 8) Devuelve las piezas sin terminar al agente planificador para su posterior iteración. Incluye scripts de análisis de código, tablas de mapeo de API, patrones de conversión de componentes y herramientas de validación automatizadas. Palabras clave desencadenantes: migrar de Android a HarmonyOS, migración de Android, migración de HarmonyOS, Java/Kotlin a ArkTS, actividad a página, comparación de funciones, comparación de UI.

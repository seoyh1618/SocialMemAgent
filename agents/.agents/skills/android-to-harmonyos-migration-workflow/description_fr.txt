Workflow de migration de code collaboratif multi-agent Android vers HarmonyOS (itératif). Déclenché lorsqu'un utilisateur doit migrer un projet Android vers HarmonyOS, fournissant : 1) L'agent analyseur analyse la structure du code 2) L'agent planificateur élabore le plan de migration 3) L'agent traducteur effectue la conversion du code 4) L'agent validateur utilise la compétence ohos-app-build-debug pour la validation de la compilation, l'empaquetage de l'application et la vérification sur le premier écran de l'appareil 5) L'agent testeur valide la fonctionnalité (exécuté après chaque migration de module) 6) Une fois tous les modules migrés, l'agent Feature Comparator vérifie les fonctionnalités manquantes. 7) L'agent de comparaison d'interface utilisateur vérifie la cohérence de l'interface utilisateur. 8) Renvoyez les pièces inachevées à l'agent de planification pour une itération ultérieure. Comprend des scripts d'analyse de code, des tables de mappage d'API, des modèles de transformation de composants et des outils de validation automatisés. Mots-clés déclencheurs : migrer Android vers HarmonyOS, migration Android, migration HarmonyOS, Java/Kotlin vers ArkTS, activité vers page, comparaison de fonctionnalités, comparaison d'interface utilisateur.

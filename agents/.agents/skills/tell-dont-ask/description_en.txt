Code review and design support based on the "Tell, Don't Ask" principle. Converts the pattern of querying an object's state and making decisions externally into one of issuing commands directly to the object. Encourages designs that strengthen encapsulation and consolidate responsibility in the objects that hold the data. Used during code reviews, new implementations, and refactoring when getter abuse or Feature Envy needs fixing. Target languages: Java, Kotlin, Scala, TypeScript, Python, Ruby, Go, Rust. Triggered by OOP design requests like "reduce getters," "improve encapsulation," "fix Feature Envy," "give responsibility to objects," or "Law of Demeter."

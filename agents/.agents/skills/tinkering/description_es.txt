Marco de experimentación seguro para agentes de IA. Crea entornos aislados de espacio aislado para probar nuevas funciones, probar enfoques y explorar soluciones sin contaminar la base de código principal.
UTILIZAR CUANDO: El agente necesita probar algo incierto, explorar múltiples enfoques, probar una nueva biblioteca, crear un prototipo de una función o ejecutar un pico técnico antes de comprometerse con la implementación.
DESENCADENADORES PRINCIPALES: "experimentar con" = Configurar zona de pruebas + ejecutar experimento "probar este enfoque" = Experimento rápido en zona de pruebas "spike" / "POC" / "prototipo" = Investigación técnica con límite de tiempo "tinker" / "modo retoques" = Ingresar al flujo de trabajo de experimentación "explorar opciones" = Comparación de múltiples enfoques en zona de pruebas
NO PARA: Depuración (use el depurador), pruebas (use el ejecutor de pruebas) o trabajo de funciones comprometidas (use ramas de git).
DIFERENCIADOR: A diferencia de las ramas de git (para dirección comprometida), retocar es para exploración "No sé si esto funcionará". Pruebe 5 cosas en la zona de pruebas antes de comprometerse con una sucursal. Comentarios más rápidos, cero contaminación del código base.

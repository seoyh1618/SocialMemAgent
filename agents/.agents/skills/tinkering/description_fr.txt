Cadre d'expérimentation sécurisé pour les agents d'IA. Crée des environnements sandbox isolés pour essayer de nouvelles fonctionnalités, tester des approches et explorer des solutions sans polluer la base de code principale.
À UTILISER QUAND : l'agent doit essayer quelque chose d'incertain, explorer plusieurs approches, tester une nouvelle bibliothèque, prototyper une fonctionnalité ou exécuter un pic technique avant de s'engager dans la mise en œuvre.
DÉCLENCHEURS PRINCIPAUX : "expérimenter avec" = configurer le bac à sable + exécuter l'expérience "essayer cette approche" = expérience rapide dans le bac à sable "spike" / "POC" / "prototype" = enquête technique limitée dans le temps "bricolage" / "mode bricolage" = entrer dans le flux de travail d'expérimentation "explorer les options" = comparaison multi-approches dans le bac à sable
PAS POUR : le débogage (utiliser le débogueur), les tests (utiliser l'exécuteur de test) ou le travail sur les fonctionnalités validées (utiliser les branches git).
DIFFÉRENTIATEUR : contrairement aux branches git (pour la direction engagée), le bricolage est destiné à l'exploration "Je ne sais pas si cela fonctionnera". Essayez 5 choses dans le bac à sable avant de vous engager dans une branche. Rétroaction plus rapide, zéro pollution de la base de code.

AI 에이전트를 위한 안전한 실험 프레임워크. 기본 코드베이스를 오염시키지 않고 새로운 기능을 시도하고, 접근 방식을 테스트하고, 솔루션을 탐색하기 위한 격리된 샌드박스 환경을 만듭니다.
사용 시기: 상담원은 구현을 시작하기 전에 불확실한 작업을 시도하고, 다양한 접근 방식을 탐색하고, 새 라이브러리를 테스트하고, 기능의 프로토타입을 제작하거나, 기술 스파이크를 실행해야 합니다.
기본 트리거: "실험" = 샌드박스 설정 + 실험 실행 "이 접근 방식 시도" = 샌드박스에서 빠른 실험 "스파이크" / "POC" / "프로토타입" = 시간 제한 기술 조사 "팅커" / "팅커링 모드" = 실험 작업 흐름 시작 "옵션 탐색" = 샌드박스의 다중 접근 방식 비교
해당되지 않음: 디버깅(디버거 사용), 테스트(테스트 실행기 사용) 또는 커밋된 기능 작업(git 브랜치 사용).
차별화 요소: git 브랜치(커밋된 방향의 경우)와 달리 Tinkering은 "이것이 작동할지 모르겠습니다" 탐색을 위한 것입니다. 브랜치에 커밋하기 전에 샌드박스에서 5가지를 시도해 보세요. 더 빠른 피드백, 제로 코드베이스 오염.

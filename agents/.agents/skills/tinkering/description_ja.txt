AI エージェントのための安全な実験フレームワーク。メインのコードベースを汚染することなく、新しい機能を試し、アプローチをテストし、ソリューションを探索するための分離されたサンドボックス環境を作成します。
使用場面: エージェントは、実装に取り​​組む前に、何か不確かなことを試したり、複数のアプローチを検討したり、新しいライブラリをテストしたり、機能のプロトタイプを作成したり、技術的なスパイクを実行したりする必要があります。
主なトリガー: 「実験」 = サンドボックスのセットアップ + 実験の実行 「このアプローチを試す」 = サンドボックスでの簡単な実験 「スパイク」 / 「POC」 / 「プロトタイプ」 = タイムボックス化された技術調査 「いじくり」 / 「いじくりモード」 = 実験ワークフローに入る 「オプションの探索」 = サンドボックスでの複数のアプローチの比較
対象外: デバッグ (デバッガーを使用)、テスト (テスト ランナーを使用)、またはコミットされた機能作業 (git ブランチを使用)。
DIFFERENTIATOR: git ブランチ (コミットされた方向のため) とは異なり、いじくり回しは「これが機能するかどうかわからない」探索のためのものです。ブランチにコミットする前に、サンドボックスで 5 つのことを試してください。フィードバックが速くなり、コードベース汚染がゼロになります。

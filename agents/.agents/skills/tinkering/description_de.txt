Sicheres Experimentier-Framework für KI-Agenten. Erstellt isolierte Sandbox-Umgebungen zum Ausprobieren neuer Funktionen, zum Testen von Ansätzen und zum Erkunden von Lösungen, ohne die Hauptcodebasis zu verschmutzen.
VERWENDEN, WENN: Der Agent muss etwas Ungewisses ausprobieren, mehrere Ansätze erkunden, eine neue Bibliothek testen, einen Prototyp einer Funktion erstellen oder einen technischen Spike ausführen, bevor er sich zur Implementierung verpflichtet.
PRIMÄRE AUSLÖSER: „Experimentieren mit“ = Sandbox einrichten + Experiment ausführen „Diesen Ansatz ausprobieren“ = Schnelles Experiment in der Sandbox „Spike“ / „POC“ / „Prototyp“ = Zeitgesteuerte technische Untersuchung „Basteln“ / „Bastelmodus“ = Experimentierworkflow starten „Optionen erkunden“ = Vergleich mehrerer Ansätze in der Sandbox
NICHT FÜR: Debuggen (Debugger verwenden), Testen (Test Runner verwenden) oder festgeschriebene Funktionsarbeit (Git Branches verwenden).
UNTERSCHEIDUNGSMERKMALE: Im Gegensatz zu Git-Zweigen (für festgelegte Richtungen) dient das Basteln der Erkundung „Ich weiß nicht, ob das funktionieren wird“. Probieren Sie fünf Dinge in der Sandbox aus, bevor Sie sich auf einen Zweig festlegen. Schnelleres Feedback, keine Verschmutzung der Codebasis.

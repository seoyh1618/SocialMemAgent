Zum Erstellen, Schreiben oder Verfeinern einer User Story oder eines Tickets verwenden. Erstellt strukturierte Geschichten mit Zweck/Überblick, funktionalen Anforderungen, Akzeptanzkriterien „Gegeben-wann-dann“ und manuellen Testanleitungen. Verwenden Sie es auch, wenn Sie aufgefordert werden, Akzeptanzkriterien zu definieren, den Umfang einer Funktion festzulegen oder eine Geschichte für die Entwicklung vorzubereiten.

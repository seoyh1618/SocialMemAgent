將 GitHub 星級儲存庫編入結構化黑曜石保險庫中
AI 合成摘要、標準化主題分類、圖形優化
wikilinks 和用於過濾視圖的 Obsidian Bases (.base) 索引檔。
透過 gh CLI 取得儲存庫元資料和自述文件，將儲存庫分類為
類別和標準化主題，產生單獨的回購註釋
frontmatter，並為類別/主題/作者建立中心註釋
作為圖形視圖連接點。

當使用者想要執行以下操作時，請使用此技能：
(1) 將他們的 GitHub star 編錄或索引到 Obsidian 中
(2) 從加星號的儲存庫建立可搜尋的知識庫
(3) 整理並發現 GitHub star 中的模式
(4) 將 GitHub star 匯出為結構化 Markdown 筆記
(5) 依主題、語言或作者建立加星號的儲存庫圖表

若要將特定 URL 儲存/擷取到筆記，請改用 kcap。
若要瀏覽 AI 推文，請使用 ai-twitter-radar。

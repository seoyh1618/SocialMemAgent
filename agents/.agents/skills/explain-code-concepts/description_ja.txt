コード内のパターン、アーキテクチャ上の選択、または設計上の決定に気づき、その背後にあるアイデアを理解したい学習者向けのガイド付きコンセプト発見。学習者がコードの読み書きはできるが、まだ名前を付けたり完全に表現したりすることのできない、より高いレベルの形状、つまりコードベース全体で繰り返される構造、アーキテクチャ上の規則、設計上のトレードオフ、または馴染みのないパラダイムを垣間見たときに使用します。 「このパターンがよく見られる」、「なぜコードベースがこのように構造化されているのか」、「ここに名前が付けられない形状がある」、「このアプローチの背後にあるアイデアは何ですか」、「すべてがこの構造に従っていることに気づきました」などのフレーズ、または学習者がそれが具体化する概念を知らずに繰り返しのパターンを説明するときにトリガーされます。

Explication de l'architecture du modèle de base du pipeline BK-CI : couvre la structure à quatre couches Pipeline/Étape/Conteneur/Tâche, la sérialisation du modèle, la gestion des versions et la validation du modèle. Utilisez-le lorsque les utilisateurs ont besoin de comprendre les structures de données du pipeline, de développer des fonctionnalités de pipeline, de gérer la conversion de modèle ou d'étendre le modèle.

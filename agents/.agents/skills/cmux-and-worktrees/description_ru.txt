Управляйте параллельной разработкой с помощью рабочих деревьев git в стиле cmux в одном репозитории. Используйте этот навык всякий раз, когда пользователь просит запустить несколько агентов параллельно, создать или возобновить изолированные рабочие деревья, составить список/переключить/объединить/удалить рабочие деревья, настроить `.cmux/setup` или восстановиться после конфликтов рабочих деревьев. Используйте команды cmux в этой среде.

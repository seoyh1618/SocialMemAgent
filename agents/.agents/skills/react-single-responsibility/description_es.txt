Estrategias para simplificar componentes, ganchos y métodos: orden de descomposición (utilidades, ganchos, subcomponentes),
retornos anticipados, flujo de control, diseño de parámetros y correcciones de olores de código. Usar cuando el usuario dice: ungodify
este método/función/componente, simplifica este método/función/componente, crea este método/función/componente
menos complejo; o al refactorizar un componente, gancho o función grande, reduciendo la complejidad, aplicando
responsabilidad única, o preguntar cómo simplificar un componente, gancho o método.

Guide les contributeurs à travers la base de code z-schema, le flux de travail des relations publiques et les tâches de développement courantes. À utiliser lorsque l'utilisateur souhaite contribuer au z-schema, ajouter une nouvelle fonctionnalité ou un nouveau mot-clé, ajouter un code d'erreur, ajouter un validateur de format, modifier des options, écrire des tests, exécuter la suite de tests, corriger un test défaillant, comprendre le pipeline de validation, naviguer dans l'architecture du code source ou soumettre une demande d'extraction. À utiliser également lorsque quelqu'un mentionne une contribution, des PR, le code source du schéma z ou l'intégration de JSON Schema Test Suite.

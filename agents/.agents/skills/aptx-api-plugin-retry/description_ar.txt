استخدم @aptx/api-plugin-retry لتنفيذ عمليات إعادة محاولة الطلب. من أجل: تكوين أعداد مرات إعادة المحاولة، وتعيين سياسات تأخير ثابتة أو ديناميكية، وتحديد مسندات retryOn (NetworkError/TimeoutError)، ومنع إعادة محاولات الطلبات غير الفعالة، وتتبع أعداد مرات إعادة المحاولة، وواجهة برمجة تطبيقات createRetryMiddleware. يتم تشغيله عند الحاجة إلى وظيفة إعادة المحاولة أو إنشاء برنامج RetryMiddleware.

Verwenden Sie @aptx/api-plugin-retry für erneute Anforderungsversuche. Für: Festlegen von Wiederholungszahlen, Konfigurieren fester oder dynamischer Verzögerungsrichtlinien, Definieren einer retryOn-Funktion (NetworkError/TimeoutError), Verhindern von Wiederholungen für nicht idempotente Anforderungen, Verfolgen von Wiederholungszahlen und der createRetryMiddleware-API. Wird ausgelöst, wenn Code eine Wiederholungsfunktion oder createRetryMiddleware benötigt.

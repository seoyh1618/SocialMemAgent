Utilice @aptx/api-plugin-retry para implementar reintentos de solicitud. Para: configurar recuentos de reintentos, establecer estrategias de retraso fijas o dinámicas, definir predicados de retryOn (NetworkError/TimeoutError), evitar reintentos para solicitudes no idempotentes, realizar un seguimiento de los recuentos de reintentos, crear la API createRetryMiddleware. Se activa cuando se necesita reintentar la lógica o createRetryMiddleware.

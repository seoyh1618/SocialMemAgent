Utilizza @aptx/api-plugin-retry per implementare i nuovi tentativi di richiesta. Utilizzato per: configurare il conteggio dei tentativi, impostare strategie di ritardo fisse o dinamiche, definire le funzioni decisionali retryOn (NetworkError/TimeoutError), impedire nuovi tentativi di richieste non idempotenti, tenere traccia dei conteggi dei tentativi e l'API createRetryMiddleware. Attivato quando il codice richiede la funzionalità di ripetizione o createRetryMiddleware.

Utilisez @aptx/api-plugin-retry pour implémenter les nouvelles tentatives de requête. Utilisé pour : configurer le nombre de tentatives, définir des stratégies de retard fixes ou dynamiques, définir les fonctions de jugement retryOn (NetworkError/TimeoutError), empêcher les tentatives de requêtes non idempotentes, suivre le nombre de tentatives et l'API createRetryMiddleware. Déclenché lorsque la fonctionnalité de nouvelle tentative ou createRetryMiddleware est nécessaire dans le code.

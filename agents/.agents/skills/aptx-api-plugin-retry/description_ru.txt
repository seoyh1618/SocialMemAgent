Используйте @aptx/api-plugin-retry для реализации повторных запросов. Он настраивает количество повторов, устанавливает стратегии фиксированной или динамической задержки, определяет предикаты retryOn (NetworkError/TimeoutError), предотвращает повторные попытки для неидемпотентных запросов, отслеживает количество повторов и предоставляет API createRetryMiddleware. Запускается, когда коду требуется функция повтора или createRetryMiddleware.

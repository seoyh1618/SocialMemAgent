Des portes de vérification de la qualité du code intégrées au cycle de vie de l'agent. Utilisez cette compétence chaque fois que vous écrivez, modifiez, révisez ou déboguez du code, y compris de nouvelles fonctionnalités, des corrections de bogues, des refactorisations, du dépannage, de la configuration CI/CD ou du démarrage de projet. À utiliser également lorsque l'utilisateur mentionne « qualité », « stratégie de test », « pipeline CI », « garde-corps », « débogage » ou demande comment améliorer la fiabilité du code. Si vous écrivez du code ou essayez de comprendre pourquoi le code ne fonctionne pas, cette compétence s'applique.

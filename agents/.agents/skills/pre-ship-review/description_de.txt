Führen Sie eine strukturierte Qualitätsprüfung durch, bevor Sie den Code an jedem Kontrollpunkt wie PRs, Releases oder Meilensteinen versenden. Verwenden Sie diese Option immer dann, wenn der Benutzer „Überprüfung vor dem Versand“, „Versandprüfung“, „Qualitätsüberprüfung“, „Selbstüberprüfung“ oder „Versandbereit“ sagt, oder vor einer wichtigen Codeübermittlung. Diese Fähigkeit erkennt Integrationsgrenzenfehler, bei denen KI-generierter Code zwischen Komponenten unterbrochen wird. NICHT für routinemäßige Linting- oder Formatierungszwecke (diese werden in Phase 2 von ITP behandelt) oder zur Überprüfung rein dokumentarischer Änderungen verwenden.

Exécutez un examen qualité structuré avant d’expédier le code à n’importe quel point de contrôle tel que les PR, les versions ou les jalons. À utiliser chaque fois que l'utilisateur dit « examen avant expédition », « contrôle d'expédition », « examen qualité », « auto-évaluation » ou « prêt à expédier », ou avant toute soumission de code importante. Cette compétence détecte les échecs des limites d'intégration lorsque le code généré par l'IA s'interrompt entre les composants. Ne PAS utiliser pour le peluchage ou le formatage de routine (ceux-ci sont traités par la phase 2 de l'ITP) ou pour réviser les modifications apportées à la documentation uniquement.

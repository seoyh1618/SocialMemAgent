Écrivez des tests unitaires idéaux et examinez ceux existants. Axé exclusivement sur les tests unitaires : teste une unité de manière isolée. Pas pour les tests d'intégration ou E2E. Deux modes : générer de nouveaux tests à partir de l'analyse du code ou examiner les tests existants pour détecter les anti-modèles et les problèmes de qualité. Indépendant de la langue, basé sur les meilleures pratiques de Beck, Fowler, Khorikov et Google.

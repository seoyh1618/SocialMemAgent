Scrivi test unitari ideali e rivedi quelli esistenti. Incentrato esclusivamente sul test unitario: testa un'unità isolatamente. Non per test di integrazione o E2E. Due modalità: generare nuovi test dall'analisi del codice o rivedere i test esistenti per antipattern e problemi di qualità. Indipendente dal linguaggio, basato sulle migliori pratiche di Beck, Fowler, Khorikov e Google.

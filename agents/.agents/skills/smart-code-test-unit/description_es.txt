Escriba pruebas unitarias ideales y revise las existentes. Centrado exclusivamente en pruebas unitarias: prueba una unidad de forma aislada. No para integración o pruebas E2E. Dos modos: generar nuevas pruebas a partir del análisis de código o revisar las pruebas existentes en busca de antipatrones y problemas de calidad. Independiente del idioma, basado en las mejores prácticas de Beck, Fowler, Khorikov y Google.

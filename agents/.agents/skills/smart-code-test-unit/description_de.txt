Schreiben Sie ideale Unit-Tests und überprüfen Sie vorhandene. Konzentriert sich ausschließlich auf Unit-Tests – testet eine Unit isoliert. Nicht für Integrations- oder E2E-Tests. Zwei Modi: Generieren Sie neue Tests aus der Codeanalyse oder überprüfen Sie vorhandene Tests auf Antimuster und Qualitätsprobleme. Sprachunabhängig, basierend auf Best Practices von Beck, Fowler, Khorikov und Google.

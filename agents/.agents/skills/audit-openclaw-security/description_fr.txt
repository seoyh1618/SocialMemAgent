Auditer et renforcer la sécurité d'OpenClaw (Gateway + agents). À utiliser lorsque l'utilisateur demande d'auditer/sécuriser/renforcer OpenClaw ; lors du dépannage d'une exposition à risque (en particulier l'interface utilisateur Web/le plan de contrôle de la passerelle sur le port 18789) ; lors de l'examen du contrôle d'accès DM/groupe (couplage/listes autorisées/gestion des mentions) ; autorisations de l'outil (exec/fs/browser/nodes/gateway/cron) ; risque lié à la chaîne d’approvisionnement des plugins/compétences ; conservation des secrets/transcriptions/journaux ; ou lors du déploiement d'OpenClaw sur un Mac mini, un ordinateur portable personnel, un hôte Docker ou une VM cloud (AWS EC2/VPS).

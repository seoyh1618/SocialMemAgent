Audite y refuerce la seguridad de OpenClaw (Gateway + agentes). Úselo cuando el usuario solicite auditar/proteger/reforzar OpenClaw; al solucionar problemas de exposición riesgosa (especialmente la interfaz de usuario web/plano de control de Gateway en el puerto 18789); al revisar el control de acceso a DM/grupo (emparejamiento/listas permitidas/bloqueo de menciones); permisos de herramientas (exec/fs/browser/nodes/gateway/cron); riesgo de la cadena de suministro de complementos/habilidades; secretos/transcripciones/retención de registros; o al implementar OpenClaw en una Mac mini, una computadora portátil personal, un host Docker o una máquina virtual en la nube (AWS EC2/VPS).

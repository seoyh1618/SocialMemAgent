Créez des problèmes GitHub avec la détection des doublons et l'analyse de la base de code. À utiliser lorsque vous découvrez un bug, une amélioration ou un problème qui ne figurait pas dans la portée/spécification d'origine lors du développement. Se déclenche sur des expressions telles que « créer un problème », « signaler un bug », « cela devrait être suivi », « nous devrions résoudre ce problème plus tard », « problème hors de portée », « remarqué un problème », « trouvé un bug ».

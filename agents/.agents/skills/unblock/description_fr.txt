Structure les tâches de codage autour du contexte MCP débloqué. Hydrate un plan avec des connaissances organisationnelles (PR, Slack, Jira, documents, historique du code), le soumet pour examen critique par rapport aux modèles d'équipe, puis génère et examine le code. Deux portes de révision obligatoires évitent les inadéquations de modèles et les roues réinventées. À utiliser pour tout changement de code : fonctionnalités, corrections de bogues, refactorisations ou enquêtes.

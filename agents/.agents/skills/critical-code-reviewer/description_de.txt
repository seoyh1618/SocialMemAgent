Führen Sie strenge, kontroverse Codeüberprüfungen ohne Toleranz für Mittelmäßigkeit durch. Wird verwendet, wenn Benutzer meinen Code oder eine PR „kritisch überprüfen“, „meinen Code kritisieren“, „Probleme in meinem Code finden“ oder „Was stimmt mit diesem Code nicht“ bitten. Identifiziert Sicherheitslücken, Lazy Patterns, Edge-Case-Fehler und schlechte Praktiken in Python, R, JavaScript/TypeScript, SQL und Front-End-Code. Untersucht die Fehlerbehandlung, Typsicherheit, Leistung, Zugänglichkeit und Codequalität. Bietet strukturiertes Feedback mit Schweregraden (Blockierung, Erforderlich, Vorschläge) und spezifischen, umsetzbaren Empfehlungen.

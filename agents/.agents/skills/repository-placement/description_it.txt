Una guida su dove posizionare le interfacce del repository. Spiega perché, in Clean Architecture, le interfacce del repository dovrebbero essere posizionate nel livello del caso d'uso piuttosto che nel livello del dominio. Si concentra sulla prevenzione dell'accoppiamento tra modelli di dominio e repository e sull'applicazione dell'intento progettuale attraverso la struttura. Attivato da richieste di posizionamento del repository come "dove posizionare il repository", "posizionamento dell'interfaccia del repository", "repository nel livello di dominio", "repository in Clean Architecture", ecc.

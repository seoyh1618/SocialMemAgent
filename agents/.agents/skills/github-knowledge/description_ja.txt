GitHub リポジトリをローカルに保存、インデックス付け、クエリするための GitHub ナレッジ ベース マネージャー。
保存されたすべてのリポジトリの概要を含む BASE.md インデックス ファイルを維持します。

ユーザーが以下に関連する質問またはリクエストを行ったときに、このスキルをトリガーします。
- 特定の GitHub リポジトリ (ダウンロード、クエリ、分析、要約、比較)
- GitHub の問題、PR、リリース、またはリポジトリの検索
- 仓库、GitHub项目、开源项目のダウンロード、询、分析等の操作

無関係な git 操作 (「ローカル git リポジトリの初期化」、「git commit」など) をトリガーしないでください。

主なアクション:
1. 明示的なダウンロード要求 → URL を要求 → クローンまたはプル → 要約 → BASE.md の更新
2. クエリ + リポジトリがローカルに存在する → プル → 概要を更新する可能性がある → 質問に答える
3. クエリ + リポジトリはローカルではありません → gh/curl 経由で GitHub を検索 → 質問に回答します (クローンは作成しないでください)

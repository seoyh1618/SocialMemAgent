Überprüfen Sie eine Codebasis auf toten Code – nicht erreichbare Funktionen, ungenutzte Importe, verwaiste Klassen, nicht erreichbare Zweige und veraltete Feature-Flags – indem Sie Live-Aufrufpfade von allen Einstiegspunkten aus verfolgen. Prüft außerdem die Korrektheit (entsprechen Implementierungen ihrem angegebenen Zweck?) und die algorithmische Solidität. Erstellt ein ASCII-Aufrufdiagramm und einen kategorisierten Bericht über toten Code. Verwenden Sie diese Option, wenn eine Codebasis viele Iterationen durchlaufen hat, ein umfangreicher Refactor durchgeführt wurde, eine Funktion entfernt wurde, der Supportcode jedoch möglicherweise bestehen bleibt, oder wenn der Benutzer „toten Code finden“, „nicht verwendeten Code bereinigen“, „überprüfen möchte, was tatsächlich verwendet wird“ oder „verwaiste Funktionen finden“ möchte.

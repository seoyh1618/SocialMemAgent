すべてのエントリ ポイントからのライブ呼び出しパスをトレースすることで、コードベースにデッド コード (到達不能な関数、未使用のインポート、孤立したクラス、到達不能なブランチ、古い機能フラグ) がないか監査します。また、正確性 (実装が主張された目的と一致しているか) とアルゴリズムの健全性も監査します。 ASCII コール グラフと分類されたデッド コード レポートを生成します。コードベースが何度も反復された場合、大規模なリファクタリングが完了した場合、機能が削除されたがそのサポート コードが残っている可能性がある場合、またはユーザーが「デッド コードの検索」、「未使用のコードのクリーンアップ」、「実際に使用されているものを監査」、または「孤立した関数の検索」を要求した場合に使用します。

Implémentez les problèmes GitHub de bout en bout à l'aide de GitHub CLI et de git, du clonage d'un référentiel (et du forking uniquement en cas de besoin) à l'ouverture d'une pull request. À utiliser lorsqu'on vous demande de récupérer un problème spécifique sur GitHub et de fournir un PR prêt à être révisé avec un contexte de problème lié, des validations, des notes de validation et une passe de qualité pré-PR via $code-review.

Implementieren Sie GitHub-Probleme durchgängig mit der GitHub-CLI und Git, vom Klonen eines Repositorys (und dem Forken nur bei Bedarf) bis zum Öffnen einer Pull-Anfrage. Verwenden Sie diese Option, wenn Sie aufgefordert werden, ein bestimmtes Problem von GitHub aufzunehmen und über $code-review eine überprüfungsbereite PR mit verknüpftem Problemkontext, Commits, Validierungsnotizen und einem Qualitätspass vor der PR bereitzustellen.

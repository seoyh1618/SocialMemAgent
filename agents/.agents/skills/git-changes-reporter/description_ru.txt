Создавайте структурированные отчеты об изменениях git (JSON + Markdown). Используйте этот навык, когда пользователь упоминает «изменения git», «сводку коммитов», «проверку кода», «примечания к выпуску», «недавние изменения», «ежедневную сводку» или ему необходимо проанализировать изменения кода для указанного диапазона коммитов. Включает семантический отчет с тройной структурой (замысел проекта, основной код, объем воздействия), подходящий для проверки кода, примечаний к выпуску, групповой синхронизации, CI/CD и других сценариев.

Examens de l'architecture dans 7 dimensions : intégrité structurelle, évolutivité, préparation de l'entreprise (SOC2/HIPAA/GDPR/PCI-DSS), performances, sécurité, excellence opérationnelle et architecture de données. Produit des rapports notés avec des recommandations hiérarchisées. Trois modes : (1) Examen de la base de code – analyse fondée sur des preuves du code source, des configurations, de l'IaC ; (2) Examen des documents — analyse basée sur les risques des documents de conception, des RFC et des spécifications ; (3) Hybride – détection de dérive entre l’intention et la mise en œuvre. Se déclenche sur : "examiner l'architecture", "critiquer la conception", "auditer le système", "évaluer la base de code", "trouver des défauts de conception", "évaluer l'évolutivité", "vérifier la sécurité", "l'état de préparation de l'entreprise", "l'évaluation de l'architecture", "la diligence raisonnable technique", ou lorsque l'utilisateur fournit un document de conception du système ou une base de code et demande des commentaires ou des améliorations.

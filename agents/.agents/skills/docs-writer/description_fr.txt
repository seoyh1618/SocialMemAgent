Générez des produits structurés et des documents techniques grâce à une découverte guidée. 8 types de documents (PRD, Brief, Issue, Task, User Story, RFC, ADR, TDD). À utiliser pour définir des produits, signaler des bogues, planifier des tâches, rédiger des histoires, proposer des changements, enregistrer des décisions, concevoir des systèmes. À utiliser également lorsque l'utilisateur souhaite documenter une idée de fonctionnalité, rédiger des exigences, formaliser une décision, décrire un bogue trouvé, planifier une architecture technique ou avoir besoin de tout document structuré pour un projet. Déclencheurs sur « créer PRD », « créer un problème », « signaler un bug », « demande de fonctionnalité », « créer une tâche », « créer une user story », « créer RFC », « créer ADR », « créer TDD », « créer un document », « écrire un document », « documenter ceci », « avoir besoin d’une spécification », « écrire des exigences ».

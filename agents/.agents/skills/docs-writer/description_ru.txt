Создавайте структурированную документацию по продуктам и техническую документацию посредством управляемых исследований. 8 типов документов (PRD, Brief, Issue, Task, User Story, RFC, ADR, TDD). Используйте при определении продуктов, сообщении об ошибках, планировании задач, написании историй, предложении изменений, записи решений, проектировании систем. Также используйте, когда пользователь хочет задокументировать идею функции, написать требования, формализовать решение, описать обнаруженную ошибку, спланировать техническую архитектуру или ему нужен какой-либо структурированный документ для проекта. Триггеры: «создать PRD», «создать проблему», «сообщить об ошибке», «запрос на функцию», «создать задачу», «создать пользовательскую историю», «создать RFC», «создать ADR», «создать TDD», «создать документ», «написать документ», «задокументировать это», «нужна спецификация», «написать требования».

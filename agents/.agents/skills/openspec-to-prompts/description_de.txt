Verwandeln Sie eine OpenSpec-Änderungsphase über eine dreistufige Pipeline in UI-bereite Build-Eingabeaufforderungen. PRD-Generierung, dann UX-Spezifikation, dann Eingabeaufforderungen zur Build-Reihenfolge. Verwenden Sie diese Option, wenn Sie mit der Implementierung einer OpenSpec-Feature-Phase beginnen oder wenn der Benutzer „openspec-to-prompts“, „Feature-Pipeline“ erwähnt oder die Konvertierung einer Spezifikationsphase in Implementierungs-Prompts anfordert. Erfordert eine OpenSpec-Änderungs-ID und eine Phasennummer als Eingabe.

A naming-convention skill to prevent getter abuse. When a getter is unavoidable in the domain model (for persistence, JSON serialization, etc.), prefix it with `breachEncapsulationOf` to flag that encapsulation is being broken. This heads off violations of the Tell-Don’t-Ask principle and deters unintentional use of the getter. Use during code reviews, new implementation, or refactoring whenever a getter is required. Target languages: Java, Kotlin, Scala, TypeScript, Python, Go, Rust. Triggered by requests about getter naming, getters that break encapsulation, persistence getters, “breachEncapsulation,” or “I need a getter but want to stop its misuse.”

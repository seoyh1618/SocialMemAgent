Generieren Sie OpenAPI/REST-Endpunkte von tRPC-Routern. Verwenden Sie diese Option, wenn Sie über eine tRPC-API verfügen, aber REST-Endpunkte für Integrationen von Drittanbietern, mobile Apps oder öffentliche API-Dokumentation verfügbar machen müssen. Bietet automatische Swagger-Benutzeroberfläche und OpenAPI-Spezifikationsgenerierung. Wählen Sie diese Option, wenn Sie typsichere interne APIs (tRPC) mit REST-Fallback für externe Verbraucher wünschen.

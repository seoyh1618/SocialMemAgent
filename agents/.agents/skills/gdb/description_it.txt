Capacità di debugger GDB per programmi C/C++. Da utilizzare quando si avvia una sessione GDB, si impostano punti di interruzione, si scorre il codice, si controllano le variabili, si esegue il debug di arresti anomali, si utilizza il debug inverso (registrazione/riproduzione), il debug remoto con gdbserver o si caricano i core dump. Si attiva su query su comandi GDB, segfault, blocchi, watchpoint, punti di interruzione condizionali, stampanti graziose, script GDB Python o debug multi-thread.

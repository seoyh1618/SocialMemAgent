Используйте @aptx/api-plugin-auth для реализации промежуточного программного обеспечения и контроллеров аутентификации на основе токенов. Поддерживает автоматическое добавление заголовка авторизации, обновление токена (активная/пассивная обработка 401), предотвращение дублирования обновлений и обратные вызовы при сбоях. Условие срабатывания: когда пользователь запрашивает функции аутентификации (настройка промежуточного программного обеспечения, обработка ошибки 401, управление токенами) или когда код включает createAuthMiddleware/createAuthController.

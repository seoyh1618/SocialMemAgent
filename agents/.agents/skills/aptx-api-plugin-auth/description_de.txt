Verwenden Sie @aptx/api-plugin-auth, um tokenbasierte Authentifizierungs-Middleware und -Controller zu implementieren. Unterstützt das automatische Hinzufügen des Authorization-Headers, die Token-Aktualisierung (aktive/passive 401-Verarbeitung), verhindert doppelte Aktualisierungen und stellt Fehlerrückrufe bereit. Auslösebedingungen: Wenn der Benutzer Authentifizierungsfunktionen anfordert (Middleware konfigurieren, 401-Aktualisierung verarbeiten, Token verwalten) oder wenn der Code createAuthMiddleware/createAuthController beinhaltet.

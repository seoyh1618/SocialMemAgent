Utilizza @aptx/api-plugin-auth per implementare middleware e controller di autenticazione token. Supporta l'aggiunta automatica dell'intestazione di autorizzazione, l'aggiornamento del token (gestione 401 attiva/passiva), la prevenzione dell'aggiornamento duplicato e i callback in caso di errore. Condizioni di attivazione: quando l'utente richiede funzionalità di autenticazione (configurare il middleware, gestire l'aggiornamento 401, gestire i token) o quando il codice coinvolge createAuthMiddleware/createAuthController.

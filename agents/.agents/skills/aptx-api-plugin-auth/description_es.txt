Utilice @aptx/api-plugin-auth para implementar controladores y middleware de autenticación de tokens. Admite la adición automática de encabezado de autorización, actualización de token (manejo 401 activo/pasivo), prevención de actualización duplicada y devolución de llamada de error. Condición de activación: cuando el usuario solicita funciones de autenticación (configurar middleware, manejar actualización 401, administrar token) o cuando el código involucra createAuthMiddleware/createAuthController.

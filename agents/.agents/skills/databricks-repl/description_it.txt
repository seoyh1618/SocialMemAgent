Eseguire il codice Python in un cluster Databricks tramite una sessione REPL con stato. Utilizzare questa competenza quando l'utente desidera eseguire Python su Databricks, eseguire analisi dei dati con Spark, addestrare modelli su un cluster, eseguire query sulle tabelle di Unity Catalog, utilizzare sub_llm() per chiamate LM ricorsive o qualsiasi attività che richieda un contesto di esecuzione Databricks persistente. Utilizza sempre una --project-dir dedicata (ad esempio, example/my-task/) per isolare session.json e repl_outputs per attività. Copre il ciclo di vita della sessione (creazione, esecuzione, attesa, annullamento, distruzione), gestione dei file di output e ripristino dell'eliminazione.

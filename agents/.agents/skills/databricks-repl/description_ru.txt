Выполняйте код Python в кластере Databricks через сеанс REPL с отслеживанием состояния. Используйте этот навык, когда пользователь хочет запустить Python на Databricks, выполнить анализ данных с помощью Spark, обучить модели в кластере, запросить таблицы каталога Unity, использовать sub_llm() для рекурсивных вызовов LM или выполнить любую задачу, требующую постоянного контекста выполнения Databricks. Всегда используйте специальный --project-dir (например, example/my-task/), чтобы изолировать session.json и repl_outputs для каждой задачи. Охватывает жизненный цикл сеанса (создание, выполнение, ожидание, отмена, уничтожение), управление выходными файлами и восстановление после выселения.

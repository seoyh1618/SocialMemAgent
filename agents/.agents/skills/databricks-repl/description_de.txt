Führen Sie Python-Code auf einem Databricks-Cluster über eine zustandsbehaftete REPL-Sitzung aus. Verwenden Sie diese Fähigkeit, wenn der Benutzer Python auf Databricks ausführen, Datenanalysen mit Spark durchführen, Modelle in einem Cluster trainieren, Unity-Katalogtabellen abfragen, sub_llm() für rekursive LM-Aufrufe verwenden oder eine andere Aufgabe ausführen möchte, die einen dauerhaften Databricks-Ausführungskontext erfordert. Verwenden Sie immer ein dediziertes --project-dir (z. B. examples/my-task/), um session.json und repl_outputs pro Aufgabe zu isolieren. Deckt den Sitzungslebenszyklus (Erstellen, Ausführen, Warten, Abbrechen, Zerstören), die Ausgabedateiverwaltung und die Wiederherstellung nach Entfernung ab.

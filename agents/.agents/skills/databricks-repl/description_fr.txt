Exécutez du code Python sur un cluster Databricks via une session REPL avec état. Utilisez cette compétence lorsque l'utilisateur souhaite exécuter Python sur Databricks, effectuer une analyse de données avec Spark, entraîner des modèles sur un cluster, interroger des tables du catalogue Unity, utiliser sub_llm() pour les appels LM récursifs ou toute tâche nécessitant un contexte d'exécution Databricks persistant. Utilisez toujours un --project-dir dédié (par exemple, examples/my-task/) pour isoler session.json et repl_outputs par tâche. Couvre le cycle de vie de la session (créer, exécuter, attendre, annuler, détruire), la gestion des fichiers de sortie et la récupération après expulsion.

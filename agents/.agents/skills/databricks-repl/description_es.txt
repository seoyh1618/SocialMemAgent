Ejecute código Python en un clúster de Databricks mediante una sesión REPL con estado. Utilice esta habilidad cuando el usuario quiera ejecutar Python en Databricks, realizar análisis de datos con Spark, entrenar modelos en un clúster, consultar tablas de Unity Catalog, usar sub_llm() para llamadas LM recursivas o cualquier tarea que requiera un contexto de ejecución persistente de Databricks. Utilice siempre un --project-dir dedicado (por ejemplo, ejemplos/mi-tarea/) para aislar session.json y repl_outputs por tarea. Cubre el ciclo de vida de la sesión (creación, ejecución, espera, cancelación, destrucción), administración de archivos de salida y recuperación de desalojo.

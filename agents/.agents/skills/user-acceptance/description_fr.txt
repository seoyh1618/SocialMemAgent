À utiliser lorsque vous terminez un ticket ou une pull request et que l'utilisateur demande à valider, démontrer ou approuver le comportement fourni, y compris les modifications non destinées à l'utilisateur. Les déclencheurs incluent « UAT », « vérifier », « me guider », « montrer ce qui a changé », « pouvons-nous fusionner ? », « signer », « test d'acceptation », « faire une démonstration », « prêt à fusionner », « valider les modifications », « montre-moi que cela fonctionne » et des expressions similaires indiquant la nécessité d'une procédure d'acceptation ou d'une démonstration avant la fusion.

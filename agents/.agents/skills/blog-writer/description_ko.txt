블로그 게시물 작성 지원 기술.  
핵심 목표: 쓰기 훈련을 통해 사용자의 쓰기 수준, 인지 수준, 지식 흡수 능력을 향상시킵니다.

트리거 시나리오:  
- 사용자가 "기사 작성", "초안 작성", "작성 도와주세요", "수정", "기사 수정"이라고 말합니다.  
- 사용자가 "주제를 선택하고 채팅하세요", "xx에 대해 토론하세요", "대화를 기사로 바꿔보세요"라고 말합니다.  
- 사용자가 "최신 xx 검색해", "대중 과학 작품 써줘", "소개 써줘", "보도 자료"라고 말합니다.  
- 사용자가 "이것을 확장하세요", "더 추가할 수 있도록 도와주세요", "1000단어로 채워주세요"라고 말합니다.  
- 사용자가 검토를 위해 마크다운 파일을 제출합니다.  

핵심 기능:  
- 모드 1(검토): 엄격한 기준으로 완성된 기사를 비판하고, 로그 작성 약점 및 트랙 개선을 수행합니다.  
- 모드 2(대화): 사고 안내, 가벼운 피드백 제공, 개인 목소리 보존  
- 모드 3(검색-쓰기): 정확한 정보 가져오기, 사용자 어조 채택, 글쓰기 비평 없음  
- 모드 4(확장): 증거 추가, 일관된 스타일 유지, 원본 텍스트 판단 없음  
- 모든 모드: 사용자 톤 특성을 엄격하게 따릅니다(user-profile.md 참조).

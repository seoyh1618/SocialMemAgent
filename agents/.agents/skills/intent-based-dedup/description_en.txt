A skill for judging code reuse based on intent (purpose) rather than literal sameness. Detects misapplications of the DRY principle—unifying code that looks the same but serves different intents—and helps decide when reuse is truly appropriate. Used when handling duplicate code during reviews, refactoring, or new implementation. Target languages: language-agnostic (Rust, Java, TypeScript, Go, Python, Kotlin, Scala, etc.). Trigger phrases: “I want to unify duplicate code,” “follow DRY,” “there’s similar code,” “merge these two functions,” “reduce code duplication,” “should this be shared,” “refactor into a common function.”

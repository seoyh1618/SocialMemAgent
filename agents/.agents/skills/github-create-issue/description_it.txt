Crea problemi GitHub con la corrispondenza intelligente dei modelli e la selezione degli strumenti. Rileva e applica automaticamente i modelli di problemi dalla directory .github/ISSUE_TEMPLATE utilizzando la corrispondenza intelligente. Sfrutta gli strumenti in ordine di precedenza (GitHub MCP Server → GitHub CLI → REST API). Da utilizzare quando è necessario creare un nuovo problema GitHub, facoltativamente con etichette, assegnatari, attività cardine o assegnazione di progetto. La capacità gestisce la corrispondenza dei modelli in modo intelligente: quando esistono più modelli, analizza la descrizione del problema per suggerire il modello migliore, chiedendo chiarimenti solo in caso di ambiguità.

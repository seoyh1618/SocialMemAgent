Corrigez un bug ou un problème spécifique dans la base de code. Prend en charge deux modes : correction immédiate ou planification d'abord. Sans arguments, exécute FIX_PLAN.md existant. Suggère toujours une couverture de test et ajoute une journalisation. À utiliser lorsque l'utilisateur dit « réparer un bug », « déboguer ceci », « quelque chose est cassé » ou colle un message d'erreur.

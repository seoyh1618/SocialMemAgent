Diese Fähigkeit sollte verwendet werden, wenn der Benutzer „Entwerfen der ArkUI-API“, „Komponenteneigenschaft hinzufügen“, „Modifikatormethode erstellen“, „ArkUI-API überprüfen“, „API verwerfen“, „JSDOC für ArkUI schreiben“ auffordert oder OpenHarmony-API-Designstandards erwähnt. Bietet umfassende Anleitungen für das API-Design von ArkUI-Komponenten gemäß den Codierungsrichtlinien von OpenHarmony, einschließlich statischer/dynamischer Schnittstellensynchronisierung, SDK-Kompilierung und Verifizierung.

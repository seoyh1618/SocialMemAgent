Erstellt Pull-Request-Beschreibungen durch Vergleich des aktuellen Zweigs mit dem übergeordneten Zweig. Erstellt semantische PR-Titel im Commit-Stil und füllt PR-Vorlagen aus. Verwenden Sie diese Option, wenn der Benutzer eine PR-Beschreibung erstellen, eine Pull-Anfrage vorbereiten oder eine Beschreibung einer Zusammenführungsanforderung erstellen möchte. Der Benutzer kann in der Anfrage Ticket-IDs (z. B. Tickets: NN-123, TB-456) aus dem Trackingsystem des Unternehmens angeben; Behandeln Sie diese als die zugehörigen Problem-IDs für die PR.

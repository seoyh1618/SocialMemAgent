Verwandeln Sie grobe Ideen in professionelle LLM-Eingabeaufforderungen. Analysiert Text, Bilder, Links und Dokumente, um mithilfe bewährter Frameworks (CoT, Few-Shot, Persona usw.) optimierte Eingabeaufforderungen zu erstellen.
VERWENDEN, WENN: Der Benutzer möchte eine Eingabeaufforderung verbessern, eine Eingabeaufforderung von Grund auf neu erstellen, eine vorhandene Eingabeaufforderung optimieren, eine vage Idee in eine strukturierte Eingabeaufforderung umwandeln, analysieren, warum eine Eingabeaufforderung nicht funktioniert, oder fragt: „Schreiben Sie mir eine Eingabeaufforderung für …“, „Diese Eingabeaufforderung verbessern“, „Diese Eingabeaufforderung erstellen“.
NICHT VERWENDEN, WENN: Der Benutzer möchte die Eingabeaufforderung selbst ausführen (führen Sie sie einfach aus), möchte allgemeine Schreibhilfe ohne Eingabeaufforderungskontext, fragt nach Code/Artikeln/Tweets (verwenden Sie stattdessen die entsprechenden Fähigkeiten) oder möchte über die Theorie der Eingabeaufforderungstechnik chatten, ohne eine Eingabeaufforderung zu erstellen.
EDGE-FÄLLE: – „Diese Eingabeaufforderung beheben“ → diese Fertigkeit (Optimierung) – „Schreiben Sie mir einen Blog-Beitrag“ → NICHT diese Fertigkeit (Inhaltserstellung, nicht Eingabeaufforderungserstellung) – „Schreiben Sie mir eine Eingabeaufforderung, die Blogbeiträge generiert“ → diese Fertigkeit – „Warum funktioniert meine Eingabeaufforderung nicht?“ → diese Fertigkeit (Diagnose + Behebung) – „اكتب لي برومبت“ → diese Fertigkeit – „حسن هالبرومبت“ → diese Fertigkeit – „اكتب لي مقال“ → NICHT diese Fertigkeit (katib-al-maqalat verwenden)
EINGABEN: Grobe Idee, vorhandene Eingabeaufforderung, Bilder, Links, Dokumente oder eine beliebige Kombination. AUSGABEN: Optimierte Eingabeaufforderung in einem Codeblock, bereit zum Kopieren. ERFOLGREICH: Die Eingabeaufforderung ist klar und strukturiert, verwendet ein geeignetes Framework und erreicht das Ziel des Benutzers.

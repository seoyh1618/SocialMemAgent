大まかなアイデアをプロレベルの LLM プロンプトに変換します。テキスト、画像、リンク、ドキュメントを分析し、実証済みのフレームワーク (CoT、Few-Shot、ペルソナなど) を使用して最適化されたプロンプトを作成します。
使用場面: ユーザーがプロンプトを改善したい場合、プロンプトを最初から作成したい場合、既存のプロンプトを最適化したい場合、漠然としたアイデアを構造化されたプロンプトに変換したい場合、プロンプトが機能しない理由を分析したい場合、または「... のプロンプトを書いてください」、「このプロンプトを改善してください」、「これをプロンプト設計してください」と尋ねる場合。
使用しない場合: ユーザーがプロンプト自体を実行したい場合 (実行するだけ)、プロンプトのコンテキストなしで一般的なライティングのヘルプが必要な場合、コード/記事/ツイートを要求する場合 (代わりに適切なスキルを使用してください)、またはプロンプトを生成せずにプロンプ​​ト エンジニアリング理論についてチャットしたい場合。
エッジケース: - 「このプロンプトを修正する」→このスキル (最適化) - 「ブログ投稿を書いてください」→このスキルではありません (プロンプト作成ではなくコンテンツ作成) - 「ブログ投稿を生成するプロンプトを書いてください」→このスキル - 「なぜ私のプロンプトが機能しないのですか?」 → このスキル (診断 + 修正) - 「اكتب لي برومبت」 → このスキル - 「حسن هالبرومبت」 → このスキル - 「اكتب لي مقال」 → このスキルではありません (katib-al-maqalat を使用)
入力: 大まかなアイデア、既存のプロンプト、画像、リンク、ドキュメント、または任意の組み合わせ。出力: コード ブロック内の最適化されたプロンプト。すぐにコピーできます。成功: プロンプトは明確で構造化されており、適切なフレームワークを使用しており、ユーザーの目標を達成しています。

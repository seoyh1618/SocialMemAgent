将粗略的想法转化为专业级的法学硕士提示。分析文本、图像、链接和文档，使用经过验证的框架（CoT、Few-Shot、Persona 等）制作优化的提示。
何时使用：用户想要改进提示、从头开始创建提示、优化现有提示、将模糊的想法转换为结构化提示、分析提示不起作用的原因，或要求“为...写一个提示”、“改进此提示”、“提示设计此”。
不要在以下情况下使用：用户想要执行提示本身（只需运行它），需要在没有提示上下文的情况下获得一般写作帮助，要求提供代码/文章/推文（改为使用适当的技能），或者想要在不产生提示的情况下讨论提示工程理论。
边缘案例： - “修复此提示” → 此技能（优化） - “给我写一篇博客文章” → 不是此技能（内容创建，而不是提示创建） - “给我写一个生成博客文章的提示” → 此技能 - “为什么我的提示不起作用？” → 此技能（诊断 + 修复）-“???” → 此技能 - “???” → 此技能 - “???” → 不是此技能（使用 katib-al-maqalat）
输入：粗略想法、现有提示、图像、链接、文档或任意组合。输出：代码块中的优化提示，准备复制。成功：提示清晰、结构化、使用适当的框架并实现用户的目标。

قم بتحويل الأفكار التقريبية إلى مطالبات LLM ذات مستوى احترافي. يحلل النصوص والصور والروابط والمستندات لصياغة مطالبات محسّنة باستخدام أطر عمل مثبتة (CoT، وFew-Shot، وPersona، وما إلى ذلك).
متى الاستخدام: يريد المستخدم تحسين مطالبة، أو إنشاء مطالبة من البداية، أو تحسين مطالبة موجودة، أو تحويل فكرة غامضة إلى مطالبة منظمة، أو تحليل سبب عدم عمل المطالبة، أو يسأل "اكتب لي مطالبة لـ..."، "تحسين هذه المطالبة"، "مطالبة بتصميم هذا".
لا تستخدم عندما: يريد المستخدم تنفيذ المطالبة بنفسه (فقط قم بتشغيلها)، أو يريد مساعدة عامة في الكتابة دون سياق سريع، أو يطلب كود/مقالات/تغريدات (استخدم المهارة المناسبة بدلاً من ذلك)، أو يريد الدردشة حول النظرية الهندسية السريعة دون إنتاج مطالبة.
حالات الحافة: - "إصلاح هذه المطالبة" ← هذه المهارة (التحسين) - "اكتب لي مشاركة مدونة" ← ليست هذه المهارة (إنشاء المحتوى، وليس إنشاء المطالبة) - "اكتب لي رسالة تنشئ منشورات مدونة" ← هذه المهارة - "لماذا لا تعمل المطالبة الخاصة بي؟" → هذه المهارة (التشخيص + الإصلاح) - "اكتب لي برومبت" → هذه المهارة - "حسن هالبرومبت" → هذه المهارة - "اكتب لي مقال" → ليس هذه المهارة (استخدم كاتب المقالات)
المدخلات: فكرة تقريبية، أو موجه موجود، أو صور، أو روابط، أو مستندات، أو أي مجموعة. المخرجات: موجه محسن في كتلة التعليمات البرمجية، جاهز للنسخ. النجاح: المطالبة واضحة ومنظمه وتستخدم الإطار المناسب وتحقق هدف المستخدم.

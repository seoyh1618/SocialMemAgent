대략적인 아이디어를 전문가 수준의 LLM 프롬프트로 변환하세요. 검증된 프레임워크(CoT, Few-Shot, Persona 등)를 사용하여 텍스트, 이미지, 링크 및 문서를 분석하여 최적화된 프롬프트를 만듭니다.
사용 시기: 사용자가 프롬프트 개선, 처음부터 프롬프트 생성, 기존 프롬프트 최적화, 모호한 아이디어를 구조화된 프롬프트로 변환, 프롬프트가 작동하지 않는 이유 분석, "...에 대한 프롬프트 작성", "이 프롬프트 개선", "이 프롬프트 엔지니어링"을 요청하려고 합니다.
다음 경우에는 사용하지 마세요: 사용자가 프롬프트 자체를 실행하기를 원하거나(그냥 실행하기를 원하거나) 프롬프트 컨텍스트 없이 일반적인 글쓰기 도움을 원하거나, 코드/기사/트윗을 요청하거나(대신 적절한 기술 사용), 프롬프트를 생성하지 않고 프롬프트 엔지니어링 이론에 대해 채팅하기를 원합니다.
특별한 사례: - "이 프롬프트 수정" → 이 기술(최적화) - "블로그 게시물 쓰기" → 이 기술 아님(메시지 생성이 아닌 콘텐츠 생성) - "블로그 게시물을 생성하는 프롬프트 작성" → 이 기술 - "왜 내 프롬프트가 작동하지 않나요?" → 이 스킬(진단 + 수정) - "اكتب لي برومبت" → 이 스킬 - "حسن هالبرومبت" → 이 스킬 - "اكتب لي مقال" → 이 스킬 아님(katib-al-maqalat 사용)
입력: 대략적인 아이디어, 기존 프롬프트, 이미지, 링크, 문서 또는 모든 조합. 출력: 코드 블록의 최적화된 프롬프트, 복사 준비 완료. 성공: 프롬프트가 명확하고 체계적이며 적절한 프레임워크를 사용하고 사용자의 목표를 달성합니다.

Support the design and implementation of First-Class Collection patterns. Guide the creation of dedicated classes that wrap collections, aggregate domain logic, and ensure immutability. Use during code reviews, new implementations, or refactoring when collection-handling logic is scattered.  
Target languages: Java, Kotlin, Scala, TypeScript, Python, Ruby, Go, Rust.  
Trigger: activates on collection-design requests such as “I want to wrap a collection,” “consolidate list logic,” “First-Class Collection,” “collection operations are scattered,” or “turn List<Order> into an Orders class.”

Разверните проект на хостинговой платформе — прочитайте YAML стека для получения точной конфигурации, определите локальные инструменты CLI (vercel, wrangler, supabase, Fly, sst), настройте базу данных, загрузите код, проверьте развертывание в реальном времени. Используйте, когда пользователь говорит «развернуть», «отправить в производство», «настроить хостинг» или после завершения /build. НЕ используйте до завершения сборки.

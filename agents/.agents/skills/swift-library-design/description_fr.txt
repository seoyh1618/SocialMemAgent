Conseils d'experts sur la conception de la bibliothèque et du framework Swift. À utiliser lorsque les développeurs mentionnent : (1) la conception d'une bibliothèque ou d'un framework Swift, (2) les modèles de conception d'API publics, (3) l'architecture orientée protocole ou les types associés, (4) les générateurs de résultats ou la conception DSL, (5) l'optimisation des performances pour les bibliothèques, (6) @inlinable ou @usableFromInline, (7) les types non copiables pour les API, (8) la divulgation progressive dans la conception de l'API, (9) les modèles ResponseGenerator ou builder.

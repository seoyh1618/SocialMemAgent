Applique le guide de style Go officiel de Google lors de l'écriture ou de la modification de fichiers Go (.go). Utilisez cette compétence AVANT de créer, de modifier ou de réviser un code Go. Couvre la dénomination (packages, récepteurs, variables, évitement des répétitions), la gestion des erreurs (structure, emballage, chaînes, erreurs sentinelles), les importations (regroupement, renommage, importations vides), l'organisation du code (taille du paquet, structure du fichier) et la documentation (commentaires de doc, conventions godoc). Déclenchez n'importe quelle tâche de code Go : nouveaux fichiers, refactors, révisions de code ou décisions d'architecture impliquant des packages Go.

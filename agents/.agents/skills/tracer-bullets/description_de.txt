Erstellt mehrschichtige Features als vertikale End-to-End-Slices anstelle von horizontalen Schichten. Jeder Abschnitt wird überprüft, bevor der nächste beginnt.

Verwenden Sie sie, wenn Sie eine Aufgabe starten, die sich über mehr als zwei Ebenen erstreckt (DB, API, UI, Tests), CRUD-Funktionen erstellen, mehrstufige Abläufe implementieren, Funktionen in Unteraufgaben zerlegen oder die Implementierungsreihenfolge planen.

Verarbeiten Sie verfügbare GitHub-Projektelemente kontinuierlich, indem Sie den Projektkontext einmal vorbereiten und dann verknüpfte Probleme in einer Schleife über den gh-issue-to-pr-Skill implementieren. Verwenden Sie es, wenn Sie aufgefordert werden, Rückstände von einem GitHub-Projektboard zu löschen, Projektprobleme stapelweise zu liefern oder wiederholt offene Projektprobleme mit gh CLI auszuwählen und zu implementieren.

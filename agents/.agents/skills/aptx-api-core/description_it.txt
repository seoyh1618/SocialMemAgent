Guida per effettuare richieste HTTP con @aptx/api-core. Utilizzato per: creare e configurare RequestClient, implementare middleware/plugin, sostituire componenti principali (trasporto/decodificatore, ecc.), gestire errori, ascoltare eventi, utilizzare Context Bag, testare estensioni. Si attiva quando il codice deve comprendere o modificare l'architettura, i meccanismi di estensione o le best practice di @aptx/api-core.

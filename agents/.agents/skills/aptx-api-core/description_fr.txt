Lignes directrices pour effectuer des requêtes HTTP avec @aptx/api-core. Utilisé pour : créer et configurer RequestClient, implémenter un middleware/plugin, remplacer les composants principaux (Transport/Décodeur, etc.), gérer les erreurs, écouter les événements, utiliser Context Bag, tester les extensions. Déclenché lorsque le code doit comprendre ou modifier l'architecture, les mécanismes d'extension et les meilleures pratiques de @aptx/api-core.

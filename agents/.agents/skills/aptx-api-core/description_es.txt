Directrices para realizar solicitudes HTTP con @aptx/api-core. Se utiliza para: crear y configurar RequestClient, implementar middleware/complementos, reemplazar componentes principales (Transporte/Decodificador, etc.), manejar errores, escuchar eventos, usar Context Bag, probar extensiones. Se activa cuando el código necesita comprender o modificar la arquitectura, los mecanismos de extensión o las mejores prácticas de @aptx/api-core.

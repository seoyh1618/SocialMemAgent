Anleitung zum Erstellen von HTTP-Anfragen mit @aptx/api-core. Verwendet für: Erstellen und Konfigurieren von RequestClient, Implementieren von Middleware/Plugins, Ersetzen von Kernkomponenten (Transport/Decoder usw.), Umgang mit Fehlern, Abhören von Ereignissen, Verwenden von Context Bag, Testen von Erweiterungen. Wird ausgelöst, wenn Code die Architektur, Erweiterungsmechanismen oder Best Practices von @aptx/api-core verstehen oder ändern muss.

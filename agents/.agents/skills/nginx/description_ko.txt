Nginx 또는 OpenResty/Lua 개발 작업을 지원할 때 이 기술을 사용하세요. 트리거에는 Nginx 구성(서버 블록, 위치 블록, 업스트림, Proxy_pass, 재작성, 맵, try_files) 작성 또는 디버깅, OpenResty/Lua 모듈 개발(콘텐츠 처리기, 액세스 제어, 헤더 조작, 공유 사전, 코소켓 사용) 및 Nginx 성능 조정(버퍼 크기 조정, 연결 유지 조정, 캐싱, 연결 풀링, 벤치마킹)이 포함됩니다. 또한 사용자가 웹 서버/역방향 프록시/CDN 컨텍스트에서 .conf 파일, nginx.conf, lua 모듈을 언급하거나 지시문 수준 Nginx 질문을 언급할 때도 사용합니다. Apache, Caddy, HAProxy 또는 Nginx와 관련 없는 일반적인 웹 서버 질문에는 사용하지 마십시오.

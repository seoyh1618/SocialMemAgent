Автоматически генерировать и добавлять строки документации ко всему измененному коду в git diff. Используйте этот навык всякий раз, когда пользователь просит: добавить строки документации к своим изменениям, задокументировать их различия в git, добавить документацию для недавних коммитов, написать строки документации для измененных/новых функций, улучшить документацию по измененным файлам или что-либо, связанное с «строкой документации» + «diff»/«изменениями»/«фиксацией». Также срабатывает, когда пользователь говорит такие вещи, как «задокументировать мои изменения», «добавить документы к тому, что я изменил» или «охват строки документации для моего PR».

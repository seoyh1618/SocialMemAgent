Générez et ajoutez automatiquement des docstrings à tout le code modifié dans un git diff. Utilisez cette compétence chaque fois que l'utilisateur demande à : ajouter des docstrings à leurs modifications, documenter leur git diff, ajouter une couverture de documentation pour les validations récentes, écrire des docstrings pour les fonctions modifiées/nouvelles, améliorer la documentation sur les fichiers modifiés, ou tout ce qui implique "docstring" + "diff"/"changes"/"commit". Déclenchez également lorsque l'utilisateur dit des choses comme « documenter mes modifications », « ajouter des documents à ce que j'ai modifié » ou « couverture docstring pour mon PR ».

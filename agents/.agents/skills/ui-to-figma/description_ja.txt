既存プロジェクトの CSS、Tailwind 設定、コンポーネントを分析してデザイン トークンを理解し、UI を視覚的に改善し、その結果を Figma MCP サーバー経由で Figma に送信します。単なる UI の調整ではなく、プロジェクトのデザイン システムに合わせた変更が保証されます。ハードコーディングされた色や間隔はなく、既存のトークンと命名ルール内での提案のみが行われます。これを使用する場合: – 「この UI を見て Figma に送信する」 / 「このページのデザインを改善して Figma に同期する」 – 「UI を分析して Figma でデザイン提案を作成する」 – Web アプリの UI を調整してデザインを Figma に同期する必要があるとき – ユーザーが「ui-to-figma」、「UI を Figma に送信する」、または「デザインを改善して Figma にプッシュする」と言ったとき 前提条件: Playwright (Web アプリのテスト スキル) と Figma MCP サーバーが利用可能である必要があります。

قم بتحليل إعدادات CSS وTailwind ومكونات مشروع حالي لفهم رموز التصميم الخاصة به، ثم قم بتحسين واجهة المستخدم بشكل مرئي وأرسل النتيجة إلى Figma عبر خادم Figma MCP. بدلاً من مجرد تعديلات على واجهة المستخدم، فهو يضمن إجراء تغييرات تتماشى مع نظام تصميم المشروع - بدون ألوان أو مسافات مرمزة، فقط مقترحات ضمن الرموز المميزة وقواعد التسمية الحالية. استخدمه عندما: - "انظر إلى واجهة المستخدم هذه وأرسلها إلى Figma" / "تحسين تصميم هذه الصفحة ومزامنتها مع Figma" - "تحليل واجهة المستخدم وإنشاء مقترح تصميم في Figma" - عندما تحتاج إلى تحسين واجهة مستخدم تطبيق ويب ومزامنة التصميم مع Figma - عندما يقول المستخدم "ui-to-figma"، أو "أرسل واجهة المستخدم إلى Figma"، أو "تحسين التصميم والدفع إلى Figma" المتطلبات الأساسية: يجب أن يكون الكاتب المسرحي (مهارة اختبار تطبيق الويب) وخادم Figma MCP متاحين.

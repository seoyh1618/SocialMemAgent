Руководство по созданию и поддержке согласованных операций на основе исходящих сообщений в Sentry. Чаще всего используется для репликации данных между хранилищами, но применим везде, где требуется конечная согласованность, включая отложенные побочные эффекты для одного хранилища, ведение журнала аудита и разветвление событий. Используйте, когда вас просят «добавить исходящий ящик», «добавить репликацию исходящего ящика», «реплицировать модель для управления хранилищем», «реплицировать модель в регион», «добавить категорию исходящих ящиков», «записать приемник сигнала исходящих сообщений», «отладить застрявшие исходящие ящики», «исходящие ящики не обрабатываются», «данные не реплицируются», «тестировать исходящие ящики», «перенести модель для использования исходящих ящиков», «обратное заполнение исходящих данных», «объединение исходящих ящиков», «RelatedRegionModel», «RelatedControlModel», «OutboxCategory», «OutboxScope» или «outbox_runner». Охватывает примеси моделей, регистрацию категорий, приемники сигналов, тестирование, заполнение и рабочие процессы отладки.

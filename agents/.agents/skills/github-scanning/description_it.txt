Modelli di raccolta dati GitHub per gli agenti del flusso di lavoro. Copre la costruzione delle query di ricerca in base all'intento, la gestione dell'intervallo di date, il restringimento dell'ambito del repository, l'integrazione di preferenze.md, l'intelligence tra repository, il modello di raccolta di flussi paralleli e il ripristino automatico per i risultati vuoti. Da utilizzare durante la creazione di agenti che cercano su GitHub problemi, PR, discussioni, rilasci, avvisi di sicurezza o stato degli elementi della configurazione.

Cree y administre especificaciones de funciones de Gherkin (archivos .feature) siguiendo el estándar Cucumber BDD. Utilice esta habilidad siempre que el usuario mencione "pepinillo", "archivo de funciones", "escenarios BDD", "especificaciones de comportamiento", "criterios de aceptación" o solicite escribir escenarios de prueba en formato Dado/Cuándo/Entonces. Funciona para cualquier proyecto en cualquier idioma, desde un repositorio nuevo hasta una base de código madura. Incluso si aún no existen archivos .feature, esta habilidad ayuda a desarrollar los primeros. Utilícelo también cuando el usuario desee revisar, organizar o ampliar las especificaciones de Gherkin existentes.

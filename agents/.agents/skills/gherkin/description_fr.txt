Créez et gérez les spécifications de fonctionnalités Gherkin (fichiers .feature) conformément à la norme Cucumber BDD. Utilisez cette compétence chaque fois que l'utilisateur mentionne « cornichon », « fichier de fonctionnalités », « scénarios BDD », « spécifications de comportement », « critères d'acceptation » ou demande d'écrire des scénarios de test au format Given/When/Then. Fonctionne pour n'importe quel projet dans n'importe quelle langue, d'un tout nouveau dépôt à une base de code mature. Même si aucun fichier .feature n’existe encore, cette compétence permet d’étayer les premiers. À utiliser également lorsque l'utilisateur souhaite réviser, organiser ou développer les spécifications Gherkin existantes.

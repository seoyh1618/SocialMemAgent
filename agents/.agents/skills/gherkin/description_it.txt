Crea e gestisci le specifiche delle funzionalità Gherkin (file .feature) seguendo lo standard Cucumber BDD. Utilizza questa abilità ogni volta che l'utente menziona "cetriolino", "file di funzionalità", "scenari BDD", "specifiche di comportamento", "criteri di accettazione" o chiede di scrivere scenari di test nel formato Dato/Quando/Allora. Funziona per qualsiasi progetto in qualsiasi lingua: da un repository nuovo di zecca a una base di codice matura. Anche se non esistono ancora file .feature, questa abilità aiuta a sostenere i primi. Utilizzare anche quando l'utente desidera rivedere, organizzare o espandere le specifiche Gherkin esistenti.

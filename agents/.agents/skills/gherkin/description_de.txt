Erstellen und verwalten Sie Gherkin-Feature-Spezifikationen (.feature-Dateien) gemäß dem Cucumber BDD-Standard. Verwenden Sie diese Fähigkeit immer dann, wenn der Benutzer „Gurke“, „Funktionsdatei“, „BDD-Szenarien“, „Verhaltensspezifikationen“, „Akzeptanzkriterien“ erwähnt oder Testszenarien im Format „Gegeben/Wann/Dann“ schreiben möchte. Funktioniert für jedes Projekt in jeder Sprache – von einem brandneuen Repo bis hin zu einer ausgereiften Codebasis. Auch wenn noch keine .feature-Dateien vorhanden sind, hilft diese Fähigkeit beim Erstellen der ersten Dateien. Verwenden Sie es auch, wenn der Benutzer vorhandene Gherkin-Spezifikationen überprüfen, organisieren oder erweitern möchte.

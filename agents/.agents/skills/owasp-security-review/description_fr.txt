Examinez le code et les architectures par rapport au Top 10 : 2025 de l'OWASP – les dix risques de sécurité des applications Web les plus critiques. À utiliser lorsque : (1) l'examen du code pour détecter les vulnérabilités de sécurité, (2) l'audit d'une fonctionnalité ou d'une base de code par rapport aux catégories OWASP, (3) la fourniture de conseils de correction pour les vulnérabilités identifiées, (4) l'écriture d'un nouveau code et la nécessité de modèles de codage sécurisés. Déclencheurs : « examen de sécurité », « audit OWASP », « vérification des vulnérabilités », « liste de contrôle de sécurité », « ce code est-il sécurisé », « examen de sécurité », « correction de la vulnérabilité ».

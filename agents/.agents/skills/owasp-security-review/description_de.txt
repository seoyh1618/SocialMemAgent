Überprüfen Sie Code und Architekturen anhand der OWASP Top 10:2025 – den zehn kritischsten Sicherheitsrisiken für Webanwendungen. Verwenden Sie, wenn: (1) Code auf Sicherheitslücken überprüft wird, (2) eine Funktion oder Codebasis anhand von OWASP-Kategorien geprüft wird, (3) Anleitungen zur Behebung identifizierter Schwachstellen bereitgestellt werden, (4) neuer Code geschrieben wird und sichere Codierungsmuster benötigt werden. Auslöser: „Sicherheitsüberprüfung“, „OWASP-Audit“, „Auf Schwachstellen prüfen“, „Sicherheitscheckliste“, „Ist dieser Code sicher“, „Sicherheitsüberprüfung“, „Schwachstelle beheben“.

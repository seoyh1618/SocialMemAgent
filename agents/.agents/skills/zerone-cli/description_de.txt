Richtlinien zur Verwendung des Zerone CLI-Toolkits. Deckt vier Hauptfunktionen ab: Generierung von API-Schnittstellencode (Zerone API), Verwaltung von Schriftartsymbolen (Zerone Font_Grabber), Gerüst für Frontend-Projekte (Create-Zerone) und Generierung von Arbeitsprotokollen (Zerone Log). Verwenden Sie diese Fähigkeit in den folgenden Szenarien: Generieren von API-Modulen, Generieren von Schnittstellencode, Aktualisieren von Schnittstellen, PNPM-API, Hinzufügen neuer Backend-Schnittstellenmodule, Iconfont-Schriftsymbole, Aktualisieren von Symbolen, PNPM-Schriftart, Symbolverwendung, Erstellen von Frontend-Projekten, PNPM-Erstellung von Zerone, Gerüstinitialisierung, tägliche/wöchentliche/monatliche Arbeitsberichte, Zerone-Protokoll. Es sollte auch dann ausgelöst werden, wenn Benutzer einfach Schlüsselwörter wie „Schnittstelle“, „Symbol“, „Neues Projekt“, „Tages-/Wochenberichte“ usw. erwähnen.

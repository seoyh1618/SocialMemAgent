Экспертное руководство по созданию готовых к использованию приложений FastAPI с модульной архитектурой, в которой каждый бизнес-домен представляет собой независимый модуль с собственными маршрутами, моделями, схемами, сервисами, кешем и миграциями. Использует UV + pyproject.toml для современного управления зависимостями Python, подкаталог имени проекта для чистой организации рабочего пространства, structlog (JSON + цветное ведение журнала), конфигурация pydantic-settings, загрузчик модулей автоматического обнаружения, асинхронная SQLAlchemy с PostgreSQL, миграция Alembic для каждого модуля, Redis/кеш памяти с пространствами имен для конкретных модулей, центральный клиент httpx, возможность наблюдения OpenTelemetry/Prometheus, отслеживание идентификатора разговора (заголовок X-Conversation-ID+cookie), условная аутентификация Keycloak/RBAC на основе приложений, принципы DDD/чистого кода и сценарии автоматизации для быстрой разработки модулей. Используйте, когда пользователь запрашивает настройку проекта FastAPI, модульную архитектуру, независимую разработку модулей, архитектуру микросервисов, асинхронные операции с базами данных, стратегии кэширования, шаблоны журналирования, управление конфигурацией, системы аутентификации, реализацию наблюдаемости или корпоративные веб-службы Python. Поддерживает максимальную глубину вложенности маршрутов 3–4, шаблоны аннулирования кэша, межмодульную связь через уровень обслуживания и комплексные рабочие процессы обработки ошибок.

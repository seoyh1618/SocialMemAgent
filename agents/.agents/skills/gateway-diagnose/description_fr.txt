Diagnostiquez les pannes de passerelle en lisant les journaux de démon, les transcriptions de session, l'état Redis et la télémétrie OTEL. Triage complet des chemins de télégramme : processus démon → canal Redis → file d'attente de commandes → session pi → API modèle → livraison de télégramme. À utiliser lorsque : "passerelle cassée", "télégramme ne fonctionne pas", "pourquoi la passerelle est-elle en panne", "la passerelle ne répond pas", "vérifier les journaux de la passerelle", "qu'est-il arrivé à la passerelle", "diagnostiquer la passerelle", "erreurs de la passerelle", "examiner les journaux de la passerelle", "repli activé", "passerelle bloquée" ou toute demande visant à comprendre pourquoi la passerelle a échoué. Différente de la compétence passerelle (opérations) : cette compétence est diagnostique.

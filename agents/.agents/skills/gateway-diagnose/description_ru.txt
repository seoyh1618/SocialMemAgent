Диагностируйте сбои шлюза, читая журналы демонов, расшифровки сеансов, состояние Redis и телеметрию OTEL. Полная сортировка пути Telegram: процесс демона → канал Redis → очередь команд → сеанс pi → API модели → доставка Telegram. Используйте, когда: «шлюз сломан», «telegram не работает», «почему шлюз не работает», «шлюз не отвечает», «проверьте журналы шлюза», «что случилось со шлюзом», «диагностика шлюза», «ошибки шлюза», «просмотр журналов шлюза», «резервный режим активирован», «шлюз завис» или при любом запросе, чтобы понять, почему шлюз не работает. В отличие от навыка шлюза (операций) — этот навык является диагностическим.

تشخيص فشل البوابة من خلال قراءة سجلات البرنامج الخفي، ونصوص الجلسة، وحالة Redis، وقياس OTEL عن بعد. الفرز الكامل لمسار Telegram: عملية البرنامج الخفي ← قناة Redis ← قائمة انتظار الأوامر ← جلسة pi ← نموذج API ← تسليم Telegram. استخدمه عندما: "البوابة معطلة"، أو "البرقية لا تعمل"، أو "لماذا البوابة معطلة"، أو "البوابة لا تستجيب"، أو "التحقق من سجلات البوابة"، أو "ما حدث للبوابة"، أو "تشخيص البوابة"، أو "أخطاء البوابة"، أو "مراجعة سجلات البوابة"، أو "تنشيط الإجراء الاحتياطي"، أو "توقف البوابة"، أو أي طلب لفهم سبب فشل البوابة. تتميز هذه المهارة عن مهارة البوابة (العمليات) بأنها تشخيصية.

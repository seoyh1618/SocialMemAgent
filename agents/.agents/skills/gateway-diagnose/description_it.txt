Diagnostica gli errori del gateway leggendo i log dei daemon, le trascrizioni delle sessioni, lo stato Redis e la telemetria OTEL. Valutazione completa del percorso di Telegram: processo daemon → canale Redis → coda di comandi → sessione pi → API del modello → consegna di Telegram. Da utilizzare quando: "gateway rotto", "telegramma non funziona", "perché il gateway non funziona", "il gateway non risponde", "controlla i registri del gateway", "cosa è successo al gateway", "diagnostica del gateway", "errori del gateway", "rivedi i registri del gateway", "fallback attivato", "gateway bloccato" o qualsiasi richiesta per capire perché il gateway non è riuscito. Distinta dalla competenza gateway (operazioni): questa competenza è diagnostica.

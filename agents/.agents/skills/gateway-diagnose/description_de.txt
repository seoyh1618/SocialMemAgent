Diagnostizieren Sie Gateway-Fehler, indem Sie Daemon-Protokolle, Sitzungsprotokolle, Redis-Status und OTEL-Telemetrie lesen. Vollständige Telegrammpfad-Triage: Daemon-Prozess → Redis-Kanal → Befehlswarteschlange → Pi-Sitzung → Modell-API → Telegrammzustellung. Wird verwendet, wenn: „Gateway defekt“, „Telegramm funktioniert nicht“, „Warum ist das Gateway ausgefallen“, „Gateway antwortet nicht“, „Gateway-Protokolle überprüfen“, „Was ist mit dem Gateway passiert“, „Gateway-Diagnose“, „Gateway-Fehler“, „Gateway-Protokolle überprüfen“, „Fallback aktiviert“, „Gateway steckt fest“ oder bei jeder Anfrage, um zu verstehen, warum das Gateway ausgefallen ist. Unterscheidet sich vom Gateway-Skill (Operationen) – dieser Skill ist diagnostisch.

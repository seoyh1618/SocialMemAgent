Implementieren Sie End-to-End-Feature-Änderungen in Go-Sphere-Gerüstprojekten, indem Sie die Sphere-Layout-Konventionen und den Generierungsworkflow befolgen. Wird zum Hinzufügen oder Ändern von APIs, Protobuf-Verträgen, Ent-Schemas, Bind/Map-Registrierung, Dienstlogik oder schichtübergreifenden Refaktoren verwendet, die das Protokoll im Vordergrund stehen und manuelle Änderungen an generierten Dateien vermeiden müssen. Diese Fähigkeit ist für jede Aufgabe ERFORDERLICH, die Go-Sphere-Protodateien, Ent-Schemata, Service-Implementierungen oder Generierungsbefehle (make gen/proto, make gen/docs, make gen/wire) betrifft.

Leichtes Brainstorming für Fehlerbehebungen, kleine Funktionen oder Codeverbesserungen, die vor der Implementierung geklärt werden müssen. Löst tiefgreifende Fragen aus, um eine genaue Ausgabe zu gewährleisten. Verwenden Sie es, wenn der Benutzer einen Fehler beheben, eine kleine Funktion hinzufügen, Code umgestalten oder gezielte Verbesserungen vornehmen möchte – und der Umfang so klein ist, dass eine vollständige Brainstorming-Sitzung nicht erforderlich ist. NICHT für große Architekturentscheidungen oder Greenfield-Projekte verwenden.

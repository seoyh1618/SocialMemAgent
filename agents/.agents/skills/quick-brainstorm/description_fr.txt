Brainstorming léger pour les corrections de bugs, les petites fonctionnalités ou les améliorations de code qui nécessitent une clarification avant la mise en œuvre. Déclenche un questionnement approfondi pour garantir un résultat précis. À utiliser lorsque l'utilisateur demande à corriger un bug, à ajouter une petite fonctionnalité, à refactoriser le code ou à apporter des améliorations ciblées - et que la portée est suffisamment petite pour qu'une session de brainstorming complète ne soit pas nécessaire. Ne PAS utiliser pour des décisions architecturales à grande échelle ou des projets entièrement nouveaux.

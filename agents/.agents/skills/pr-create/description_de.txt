Erstellt eine Pull-Anfrage aus aktuellen Änderungen, überwacht GitHub CI und debuggt alle Fehler, bis CI erfolgreich ist. Verwenden Sie dies, wenn der Benutzer „PR erstellen“, „PR erstellen“, „Pull-Anfrage öffnen“, „PR senden“, „PR für diese Änderungen“ sagt oder seine aktuelle Arbeit in eine überprüfbare PR umwandeln möchte. Geht davon aus, dass das Projekt Git verwendet, auf GitHub gehostet wird und über GitHub Actions CI mit automatisierten Prüfungen (Lint, Build, Tests usw.) verfügt. Führt NICHT zusammen – stoppt, wenn CI durchläuft und stellt den PR-Link bereit.

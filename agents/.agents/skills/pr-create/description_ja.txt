現在の変更からプル リクエストを作成し、GitHub CI を監視し、CI が合格するまで障害をデバッグします。ユーザーが「pr を作成する」、「pr を作成する」、「プル リクエストを開く」、「pr を送信する」、「これらの変更について pr を送信する」と言う場合、または現在の作業をレビュー可能な PR にまとめたい場合にこれを使用します。プロジェクトが git を使用し、GitHub でホストされ、自動チェック (lint、ビルド、テストなど) を備えた GitHub Actions CI があることを前提としています。マージしません - CI が通過すると停止し、PR リンクを提供します。

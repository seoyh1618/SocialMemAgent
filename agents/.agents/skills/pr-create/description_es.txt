Crea una solicitud de extracción a partir de los cambios actuales, monitorea GitHub CI y depura cualquier falla hasta que se apruebe el CI. Utilice esto cuando el usuario diga "crear pr", "hacer un pr", "abrir solicitud de extracción", "enviar pr", "pr para estos cambios" o quiera incluir su trabajo actual en un PR revisable. Se supone que el proyecto usa git, está alojado en GitHub y tiene GitHub Actions CI con comprobaciones automatizadas (lint, build, pruebas, etc.). NO se fusiona: se detiene cuando CI pasa y proporciona el enlace PR.

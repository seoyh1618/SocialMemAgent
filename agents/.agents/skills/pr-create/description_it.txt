Crea una richiesta pull dalle modifiche correnti, monitora l'elemento CI GitHub ed esegue il debug di eventuali errori finché l'elemento CI non viene superato. Utilizzalo quando l'utente dice "crea pr", "crea un pr", "apri richiesta pull", "invia pr", "pr per queste modifiche" o desidera inserire il proprio lavoro corrente in un PR rivedibile. Presuppone che il progetto utilizzi git, sia ospitato su GitHub e disponga di GitHub Actions CI con controlli automatizzati (lint, build, test, ecc.). NON si unisce: si ferma quando CI passa e fornisce il collegamento PR.

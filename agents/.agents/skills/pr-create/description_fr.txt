Crée une demande d'extraction à partir des modifications actuelles, surveille GitHub CI et débogue tout échec jusqu'à ce que CI passe. Utilisez-le lorsque l'utilisateur dit "créer un pr", "créer un pr", "ouvrir une demande d'extraction", "soumettre un pr", "pr pour ces modifications", ou souhaite intégrer son travail actuel dans un PR révisable. Suppose que le projet utilise git, est hébergé sur GitHub et dispose de GitHub Actions CI avec des vérifications automatisées (lint, build, tests, etc.). Ne fusionne PAS - s'arrête lorsque CI passe et fournit le lien PR.

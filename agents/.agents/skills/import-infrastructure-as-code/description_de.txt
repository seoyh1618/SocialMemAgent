Importieren Sie vorhandene Azure-Ressourcen mithilfe der Azure CLI-Erkennung und Azure Verified Modules (AVM) in Terraform. Verwenden Sie es, wenn Sie aufgefordert werden, eine Live-Azure-Infrastruktur zurückzuentwickeln, Infrastruktur als Code aus vorhandenen Abonnements/Ressourcengruppen/Ressourcen-IDs zu generieren, Abhängigkeiten zuzuordnen, genaue Importadressen aus der heruntergeladenen Modulquelle abzuleiten, Konfigurationsdrift zu verhindern und AVM-basierte Terraform-Dateien zu erstellen, die für die Validierung und Planung für jeden Azure-Ressourcentyp bereit sind.

Entscheidungsframework für die Local-First-Architektur für Webanwendungen. Behandelt, wann man auf lokal zuerst vs. serverbasiert vs. hybrid umsteigen sollte, die Auswahl der Synchronisierungs-Engine (ElectricSQL, Zero, PowerSync, Replicache, LiveStore, Triplit), clientseitige Speicheroptionen (IndexedDB, OPFS, SQLite WASM, PGlite) und Konfliktlösungsstrategien (LWW, CRDTs, Server-Siege, Zusammenführung auf Feldebene).

Verwenden Sie diese Option, wenn Sie entscheiden, ob Sie eine Local-First-Architektur übernehmen möchten, eine Synchronisierungs-Engine auswählen, Client-Speicher auswählen oder Konfliktlösungsstrategien entwerfen möchten.

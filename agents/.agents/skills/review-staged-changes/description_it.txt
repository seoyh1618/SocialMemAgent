Esamina le modifiche git graduali per qualità del codice, manutenibilità, leggibilità e potenziali regressioni. Verifica che le modifiche abbiano senso nel contesto, migliorino la manutenibilità, la leggibilità e non introducano effetti collaterali. Da utilizzare durante la revisione delle modifiche in fasi, l'esame delle differenze git o quando l'utente chiede di rivedere le modifiche prima di confermarle.

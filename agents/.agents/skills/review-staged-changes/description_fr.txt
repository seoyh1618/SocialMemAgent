Les examens ont organisé les modifications de Git pour la qualité du code, la maintenabilité, la lisibilité et les régressions potentielles. Vérifie que les changements ont du sens dans le contexte, améliorent la maintenabilité, améliorent la lisibilité et n'introduisent pas d'effets secondaires. À utiliser lors de la révision des modifications par étapes, de l'examen des différences git ou lorsque l'utilisateur demande de réviser les modifications avant de les valider.

Организует постепенную очистку учетной записи GitHub с помощью шестиэтапного процесса аудита→утверждения→выполнения, который предотвращает случайное удаление. ПЕРЕД любым деструктивным действием репо, вызовите FIRST — отслеживает оповещения Dependabot о неиспользуемых прямых задержках (удаление) или только транзитивных (обновление файла блокировки). Срабатывает при «очистке GitHub», «проверке моих репозиториев», «проблемах с Dependabot», «неиспользуемых данных», «устаревших вилках», «аудите зависимостей». Требуется gh CLI. (пользователь)

Reagieren Sie auf Muster zur Leistungsoptimierung, einschließlich Memoisierung, Codeaufteilung, Reduzierung der Bundle-Größe, Eliminierung erneuter Renderings und Profilerstellung. Behandelt die automatische Optimierung des React Compilers, das manuelle Memo/useMemo/useCallback-Targeting, React.lazy mit Suspense, die Vermeidung von Barrel-Dateien, die Sichtbarkeit von Inhalten für große Listen, startTransition für nicht dringende Aktualisierungen und die Profilerstellung von React DevTools.

Verwenden Sie diese Option, um die Leistung der React-App zu optimieren, die Bundle-Größe zu reduzieren, unnötige erneute Renderings zu vermeiden, langsame Komponenten zu debuggen, Code aufzuteilen oder Rendering-Engpässe zu profilieren. Verwendung für Leistungsprüfung, Bundle-Analyse, Re-Rendering-Diagnose, verzögertes Laden und Virtualisierung.

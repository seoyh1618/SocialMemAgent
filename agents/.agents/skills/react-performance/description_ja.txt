React のパフォーマンス最適化パターンには、メモ化、コード分割、バンドル サイズの削減、再レンダリングの排除、プロファイリングが含まれます。 React Compiler の自動最適化、手動メモ/useMemo/useCallback ターゲット、Suspense を使用した React.lazy、バレル ファイルの回避、大きなリストのコンテンツの可視性、緊急ではない更新の startTransition、および React DevTools プロファイリングについて説明します。

React アプリのパフォーマンスの最適化、バンドル サイズの削減、不必要な再レンダリングの排除、遅いコンポーネントのデバッグ、コード分割、またはレンダリングのボトルネックのプロファイリングを行う場合に使用します。パフォーマンス監査、バンドル分析、再レンダリング診断、遅延読み込み、仮想化に使用します。

Escriba pruebas E2E ideales y revise las existentes. Centrado exclusivamente en pruebas de un extremo a otro: recorridos completos del usuario a través de un navegador real. No para pruebas unitarias o de integración. Dos modos: generar nuevas pruebas E2E a partir del análisis de características o revisar las pruebas E2E existentes en busca de antipatrones, descamación y problemas de calidad. Independiente del marco, basado en las mejores prácticas de Beck, Fowler, Cohn, Rauch y Google.

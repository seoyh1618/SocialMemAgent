Rédigez des tests E2E idéaux et examinez ceux existants. Axé exclusivement sur les tests de bout en bout : des parcours utilisateur complets via un véritable navigateur. Pas pour les tests unitaires ou d'intégration. Deux modes : générez de nouveaux tests E2E à partir de l'analyse des fonctionnalités ou examinez les tests E2E existants pour détecter les anti-modèles, les défauts et les problèmes de qualité. Indépendant du framework, basé sur les meilleures pratiques de Beck, Fowler, Cohn, Rauch et Google.

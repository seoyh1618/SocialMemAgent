Configura le variabili di ambiente e le impostazioni del progetto utilizzando mise [env] come unica fonte di verità. Da utilizzare ogni volta che l'utente ha bisogno di impostare mise.toml, centralizzare le variabili di ambiente, configurare venvs Python tramite mise, utilizzare modelli mise o strutturare configurazioni hub-spoke e monorepo mise con sostituzioni di sottocartelle. NON utilizzare per l'orchestrazione delle attività Mise (utilizzare invece Mise-task) o per la gestione della versione runtime non correlata alla configurazione dell'ambiente.

Persistentes Entscheidungsprotokoll, das eine Lese-/Schreib-Feedbackschleife erstellt: Lesen Sie frühere Entscheidungen, bevor Sie mit der Arbeit beginnen, respektieren oder ersetzen Sie sie und schreiben Sie dabei neue Y-Anweisungsdatensätze. Wird bei jeder Aufgabe aktiviert, die Architektur-, Struktur- oder Designentscheidungen betrifft – Bibliotheken auswählen, Schemata entwerfen, Module erstellen, Kompromisse eingehen, Muster auswählen. Wird auch aktiviert, wenn DECISIONS.md bereits im Projekt vorhanden ist. Bei geringfügigen Änderungen wie Tippfehlern, Umbenennungen oder Formatierungen NICHT aktivieren.

Разработка Flutter для экранов, функций и бизнес-логики.
ИСПОЛЬЗУЙТЕ КОГДА: создание экранов, реализация навигации, добавление AppServices,
исправление ошибок, управление состоянием, разрешения RBAC, многошаговые формы.
НЕ ДЛЯ: отдельных компонентов пользовательского интерфейса <300 строк (используйте компоненты flutter-ui).

Примеры:
<пример>
Контекст: пользователю нужна полноценная многоэкранная функция.
user: "Реализовать процесс бронирования отелей с поиском, деталями и подтверждением"
помощник: «Для этого полного процесса с навигацией и состоянием я воспользуюсь навыками Flutter-разработчика».
<commentary>Для многоэкранных потоков требуется flutter-developer.</commentary>
</пример>
<пример>
Контекст: необходим простой компонент пользовательского интерфейса.
user: "Создайте новый компонент кнопки действия"
помощник: «Для этого единственного компонента я буду использовать компоненты flutter-ui».
<commentary>Отдельные компоненты используют компоненты flutter-ui, а не flutter-developer.</commentary>
</пример>

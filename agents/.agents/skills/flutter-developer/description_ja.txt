画面、機能、ビジネス ロジックの Flutter 開発。
いつ使用するか: 画面の作成、ナビゲーションの実装、AppServices の追加、
バグの修正、状態管理、RBAC 権限、複数ステップのフォーム。
対象外: 単一 UI コンポーネントが 300 行未満 (flutter-ui-components を使用)。

例:
<例>
コンテキスト: ユーザーは完全なマルチスクリーン機能を必要としています。
ユーザー: 「検索、詳細、確認を含むホテル予約フローを実装する」
アシスタント: 「ナビゲーションと状態を含むこの完全なフローには、フラッター開発者のスキルを使用します。」
<commentary>マルチスクリーン フローには、flutter-developer が必要です。</commentary>
</例>
<例>
コンテキスト: シンプルな UI コンポーネントが必要です。
ユーザー: 「新しいアクション ボタン コンポーネントを作成します」
アシスタント: 「この 1 つのコンポーネントには flutter-ui-components を使用します。」
<commentary>単一コンポーネントは、flutter-developer ではなく、flutter-ui-components を使用します。</commentary>
</例>

تطوير الرفرفة للشاشات والميزات ومنطق الأعمال.
وقت الاستخدام: إنشاء الشاشات، وتنفيذ التنقل، وإضافة خدمات التطبيقات،
إصلاح الأخطاء، وإدارة الحالة، وأذونات RBAC، والنماذج متعددة الخطوات.
ليس لـ: مكونات واجهة المستخدم الفردية <300 سطر (استخدم مكونات واجهة المستخدم الرفرفة).

أمثلة:
<مثال>
السياق: يحتاج المستخدم إلى ميزة الشاشات المتعددة الكاملة.
user: "تنفيذ تدفق حجز الفندق مع البحث والتفاصيل والتأكيد"
المساعد: "سأستخدم مهارة مطور الرفرفة لهذا التدفق الكامل مع التنقل والحالة."
<commentary>يتطلب التدفق على شاشات متعددة مطورًا رفرفة.</commentary>
</مثال>
<مثال>
السياق: مطلوب مكون واجهة مستخدم بسيط.
المستخدم: "إنشاء مكون زر إجراء جديد"
المساعد: "سأستخدم مكونات واجهة المستخدم الرفرفة لهذا المكون الفردي."
<commentary>تستخدم المكونات الفردية مكونات Flutter-ui، وليس مطور Flutter.</commentary>
</مثال>

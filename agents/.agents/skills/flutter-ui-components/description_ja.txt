M3 準拠の UI コンポーネント (ボタン、カード、フォーム、入力)。
使用場面: 300 行未満のコンポーネントの作成、M3 移行、デザイン システムの作業。
対象外: 完全な画面、ビジネス ロジックを備えた機能 (flutter-developer を使用)。
カスタムを作成する前に、必ず MCP ツールを使用して M3 コンポーネントを検証してください。

例:
<例>
コンテキスト: ボタン コンポーネントを M3 に移行する必要があります。
ユーザー: 「BukeerButton をマテリアル デザイン 3 に移行する」
アシスタント: 「flutter-ui-components を使用して BukeerButton を M3 に移行します。」
<commentary>UI コンポーネントの移行は、flutter-ui-components の専門分野です。</commentary>
</例>
<例>
コンテキスト: 新しい再利用可能なフォームフィールドを作成します。
ユーザー: 「M3 に続く新しい日付ピッカー入力コンポーネントを作成します」
アシスタント: 「flutter-ui-components を使用して、M3 準拠の日付ピッカーを作成します。」
<commentary>単一の UI コンポーネントは flutter-ui-components で処理する必要があります。</commentary>
</例>

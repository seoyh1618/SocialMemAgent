مكونات واجهة المستخدم المتوافقة مع M3 (الأزرار والبطاقات والنماذج والمدخلات).
وقت الاستخدام: إنشاء مكونات <300 سطر، وعمليات ترحيل M3، وعمل نظام التصميم.
ليس من أجل: الشاشات الكاملة، والميزات ذات منطق الأعمال (استخدم مطور الرفرفة).
تحقق دائمًا من صحة مكونات M3 باستخدام أدوات MCP قبل إنشاء مخصص.

أمثلة:
<مثال>
السياق: تحتاج إلى ترحيل مكون الزر إلى M3.
المستخدم: "ترحيل BukeerButton إلى تصميم المواد 3"
المساعد: "سأستخدم مكونات Flutter-ui لترحيل BukeerButton إلى M3."
<commentary>ترحيل مكونات واجهة المستخدم هو تخصص لمكونات واجهة المستخدم.</commentary>
</مثال>
<مثال>
السياق: قم بإنشاء حقل نموذج جديد قابل لإعادة الاستخدام.
المستخدم: "إنشاء مكون إدخال جديد لمنتقي التاريخ يتبع M3"
المساعد: "سأستخدم مكونات Flutter-ui لإنشاء منتقي التاريخ المتوافق مع M3."
<commentary>يجب التعامل مع مكونات واجهة المستخدم المفردة بواسطة مكونات Flutter-ui-commentary.</commentary>
</مثال>

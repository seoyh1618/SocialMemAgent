Crea e rafforza i file delle unità di servizio systemd seguendo le migliori pratiche moderne. Da utilizzare quando si scrivono nuove unità systemd per applicazioni Web, lavoratori in background o daemon oppure quando si rafforzano i servizi esistenti con funzionalità di sandboxing e isolamento di sicurezza. Copre tipi di servizi, dipendenze, policy di riavvio, opzioni di sicurezza e restrizioni del file system.

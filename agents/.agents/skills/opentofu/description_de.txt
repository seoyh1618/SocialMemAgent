Infrastruktur als Code mit OpenTofu (Open-Source-Terraform-Fork) und Pulumi. Behandelt OpenTofu HCL-Syntax, Anbieter, Ressourcen, Datenquellen, Module, Statusverwaltung mit Remote-Backends, Arbeitsbereiche, Importieren vorhandener Infrastruktur, Workflow planen/anwenden, Variablenverwaltung, Ausgabewerte, Bereitsteller und Statusverschlüsselung (exklusiv für OpenTofu). Enthält Pulumi TypeScript/Python SDKs, Stack-Management, Komponentenressourcen, Konfiguration/Geheimnisse, Status-Backends, Richtlinie als Code und Automatisierungs-API. Gängige Muster für Multi-Umgebungs-Setups, Modulzusammensetzung, CI/CD-Integration, Drifterkennung und Geheimverwaltung.

Verwenden Sie es beim Schreiben oder Überprüfen von HCL-Konfigurationen, beim Verwalten des Cloud-Infrastrukturstatus, beim Migrieren von Terraform zu OpenTofu, beim Erstellen von Pulumi-Programmen in TypeScript oder Python, beim Einrichten von IaC-Pipelines für mehrere Umgebungen oder beim Implementieren der Statusverschlüsselung.

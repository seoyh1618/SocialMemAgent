プロンプト、入力フォルダー、またはモック データを含む要件ドキュメントから、go-sphere スキャフォールド プロジェクトの proto3 + HTTP API コントラクトを設計します。サービス API を定義する場合、entpb/共有メッセージ/カスタム メッセージのいずれかを選択する場合、およびスキャフォールド規則、ルーターの安全性ルール、およびサービス ローカルのエラー配置を強制する場合に使用します。このスキルは、go-sphere scaffold の proto API 設計タスクに必須です。proto ファイルを最初から作成するのではなく、常にこのスキルを使用してください。

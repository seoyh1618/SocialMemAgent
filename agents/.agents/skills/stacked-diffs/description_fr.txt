Divisez les modifications de code importantes en petites requêtes d'extraction empilées à l'aide de Vanilla git et de la CLI gh. Déclenchement automatique lors de la mise en œuvre d'une fonctionnalité ou d'une modification qui s'étend sur plusieurs étapes logiques, touche plusieurs fichiers ou dépasse environ 200 lignes modifiées. Déclenchez également sur « pile PR », « divisez cela en PR plus petits », « diffs empilés » ou « créez une pile PR ». Ne déclenchez PAS les corrections d'un seul fichier, les petites corrections de bogues ou les modifications inférieures à environ 200 lignes qui constituent une seule unité logique.

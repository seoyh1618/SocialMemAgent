Divida los cambios de código grandes en solicitudes de extracción pequeñas y apiladas utilizando Vanilla Git y la CLI de gh. Activación automática al implementar una función o cambio que abarca varios pasos lógicos, afecta a varios archivos o supera las ~200 líneas modificadas. También active "apilar PR", "dividir esto en PR más pequeños", "diferencias apiladas" o "crear una pila de PR". NO active las correcciones de un solo archivo, pequeñas correcciones de errores o cambios menores de ~200 líneas que sean una sola unidad lógica.

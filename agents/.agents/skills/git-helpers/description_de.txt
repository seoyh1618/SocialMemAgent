Git-Workflow-Helfer für herkömmliche Commits, Codeüberprüfung mit Vertrauensbewertung und Pull-Request-Management. Wird zum Festschreiben von Änderungen, Überprüfen von Code, Erstellen von PRs und PR-Beschreibungen verwendet. Verwenden Sie dies auch, wenn der Benutzer sagt, dass er mit dem Codieren fertig ist und zum Commit bereit ist, Feedback zu seinen Änderungen möchte, eine PR pushen und öffnen muss oder darum bittet, seinen Code vor dem Zusammenführen zu überprüfen. Löst aus bei „Commit“, „Review“, „Push“, „PR erstellen“, „PR-Beschreibung“, „Änderungen zusammenfassen“, „Fertig“, „Bereit zum Push“, „Meinen Code prüfen“.

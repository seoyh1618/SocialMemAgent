Unterteilt eine PRD mithilfe vertikaler Slices (Tracer-Bullets) in unabhängig erfassbare GitHub-Probleme. Verwenden Sie diese Option, wenn Sie eine PRD in Implementierungsarbeit zerlegen, Tracer-Bullet-Probleme erstellen oder wenn der Benutzer „PRD aufschlüsseln“, „Probleme aus PRD erstellen“ oder „/prd-to-issues“ sagt.

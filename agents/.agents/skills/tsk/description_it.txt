Utilizza questa competenza ogni volta che l'utente desidera tenere traccia delle attività, gestire il backlog di un progetto, creare elenchi di cose da fare o organizzare elementi di lavoro utilizzando l'archiviazione YAML basata su file. I trigger includono: "monitora questa attività", "aggiungi un'attività", "mostra le mie attività", "qual è lo stato", "sposta l'attività in completata", "riepilogo del progetto", "elenco delle attività", "backlog", "crea un tracker delle attività", "inizializza il monitoraggio del progetto", menzioni della directory .tsk o qualsiasi richiesta di gestire elementi di lavoro con priorità, stati, tag o registri di attività. Da utilizzare anche quando l'utente richiede di impostare una gestione leggera del progetto senza servizi esterni o quando si lavora in un repository che dispone già di una directory .tsk/.

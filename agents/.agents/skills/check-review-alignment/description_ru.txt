Используйте только тогда, когда пользователь явно запрашивает «проверить/оптимизировать цитаты в теле обзора `{topic}_review.tex`» или «запустить check-review-alignment».

С помощью семантического понимания хост-ИИ проверяйте каждую цитату на соответствие исходному контенту; **только при обнаружении фатальной ошибки цитирования** выполнить минимальную переписывание «предложения, содержащего цитату», и повторно использовать сценарий рендеринга `systematic-literature-review` для вывода PDF/Word (сам сценарий не вызывает локальный API LLM).

Основной принцип: **не меняйтесь ради изменения**. Если вы не уверены, является ли это фатальной ошибкой, оставьте все как есть и отметьте предупреждение в отчете.

⚠️ Не применимо для:
- Пользователи, которые просто хотят создать текст систематического обзора (используйте систематический обзор литературы).
- Пользователи, которые хотят только добавлять/проверять записи BibTeX (используйте специальный рабочий процесс управления нагрудниками)

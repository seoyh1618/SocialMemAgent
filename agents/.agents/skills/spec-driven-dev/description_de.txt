Spezifikationsgesteuerter Entwicklungsworkflow: Anforderungen → Design → Aufgaben → Implementierungsschleife. Verwenden Sie es, wenn Sie Funktionen planen, anhand von Spezifikationen implementieren oder die Arbeit wieder aufnehmen. Befehle: spec-plan, spec-go, spec-task, spec-audit, spec-status, spec-merge, spec-reset, spec-help. WICHTIG: Bearbeiten Sie niemals „requirements.md“, „design.md“, „tasks.md“ oder „progress.txt“ ohne zuvor diesen Skill und die relevanten Referenzen/Dokumente gelesen zu haben.

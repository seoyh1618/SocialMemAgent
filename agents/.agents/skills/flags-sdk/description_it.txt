Da utilizzare quando: (1) Creazione o dichiarazione di flag di funzionalità con `flag()`, (2) Impostazione di fornitori/adattatori di flag di funzionalità (Vercel, Statsig, LaunchDarkly, PostHog, GrowthBook, Hypertune, Edge Config, OpenFeature, Flagsmith, Reflag, Split, Optimizely o adattatori personalizzati), (3) Implementazione di modelli di precalcolo per pagine statiche con flag di funzionalità, (4) Impostazione del contesto di valutazione con "identify" e "dedupe", (5) Integrazione di Flags Explorer/Vercel Toolbar, (6) Utilizzo dei flag di funzionalità in Next.js (App Router, Pages Router, Middleware) o SvelteKit, (7) Scrittura di adattatori personalizzati, (8) Crittografia/decrittografia dei valori dei flag per la barra degli strumenti, (9) Qualsiasi attività che coinvolga `flags`, `flags/next`, `flags/sveltekit`, `flags/react` o pacchetti `@flags-sdk/*`. Trigger su: flag di funzionalità, test A/B, sperimentazione, SDK di flag, adattatori di flag, flag di precalcolo, Flags Explorer, gate di funzionalità, override di flag.

Guía completa para implementar indicadores de funciones y pruebas A/B utilizando el SDK de Flags (el paquete npm `flags`). Úselo cuando: (1) Crear o declarar indicadores de funciones con `flag()`, (2) Configurar proveedores/adaptadores de indicadores de funciones (Vercel, Statsig, LaunchDarkly, PostHog, GrowthBook, Hypertune, Edge Config, OpenFeature, Flagsmith, Reflag, Split, Optimizely o adaptadores personalizados), (3) Implementar patrones de precómputo para páginas estáticas con indicadores de funciones, (4) Configurar el contexto de evaluación con `identify` y `dedupe`, (5) Integrar Flags Explorer/Barra de herramientas Vercel, (6) Trabajar con indicadores de funciones en Next.js (App Router, Pages Router, Middleware) o SvelteKit, (7) Escribir adaptadores personalizados, (8) Cifrar/descifrar valores de indicadores para la barra de herramientas, (9) Cualquier tarea que involucre `flags`, `flags/next`, `flags/sveltekit`, `flags/react`, o paquetes `@flags-sdk/*`. Activadores de: indicadores de funciones, pruebas A/B, experimentación, SDK de indicadores, adaptadores de indicadores, indicadores de precómputo, Explorador de indicadores, puertas de funciones, anulaciones de indicadores.

使用 Flags SDK（“flags” npm 包）实现功能标志和 A/B 测试的综合指南。在以下情况下使用：(1) 使用“flag()”创建或声明功能标志，(2) 设置功能标志提供程序/适配器（Vercel、Statsig、LaunchDarkly、PostHog、GrowthBook、Hypertune、Edge Config、OpenFeature、Flagsmith、Reflag、Split、Optimizely 或自定义适配器），(3) 使用功能标志实现静态页面的预计算模式，(4) 使用“identify”和设置评估上下文`dedupe`，(5) 集成 Flags Explorer / Vercel 工具栏，(6) 在 Next.js（App Router、Pages Router、Middleware）或 SvelteKit 中使用功能标志，(7) 编写自定义适配器，(8) 加密/解密工具栏的标志值，(9) 涉及 `flags`、`flags/next`、`flags/sveltekit`、`flags/react` 的任何任务，或`@flags-sdk/*` 包。触发：功能标志、A/B 测试、实验、标志 SDK、标志适配器、预计算标志、标志资源管理器、功能门、标志覆盖。

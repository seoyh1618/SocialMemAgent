Guide complet pour implémenter des indicateurs de fonctionnalités et des tests A/B à l'aide du SDK Flags (le package npm `flags`). À utiliser lorsque : (1) Création ou déclaration d'indicateurs de fonctionnalités avec `flag()`, (2) Configuration de fournisseurs/adaptateurs d'indicateurs de fonctionnalités (Vercel, Statsig, LaunchDarkly, PostHog, GrowthBook, Hypertune, Edge Config, OpenFeature, Flagsmith, Reflag, Split, Optimizely ou adaptateurs personnalisés), (3) Implémentation de modèles de précalcul pour les pages statiques avec des indicateurs de fonctionnalité, (4) Configuration du contexte d'évaluation avec `identify` et `dedupe`, (5) Intégration de la barre d'outils Flags Explorer / Vercel, (6) Utilisation des indicateurs de fonctionnalités dans Next.js (App Router, Pages Router, Middleware) ou SvelteKit, (7) Écriture d'adaptateurs personnalisés, (8) Cryptage/déchiffrement des valeurs d'indicateur pour la barre d'outils, (9) Toute tâche impliquant les `flags`, `flags/next`, `flags/sveltekit`, Packages `flags/react` ou `@flags-sdk/*`. Déclencheurs sur : les indicateurs de fonctionnalités, les tests A/B, l'expérimentation, le SDK d'indicateurs, les adaptateurs d'indicateurs, les indicateurs de précalcul, l'explorateur d'indicateurs, les portes de fonctionnalités, les remplacements d'indicateurs.

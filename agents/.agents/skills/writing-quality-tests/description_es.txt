Guía para escribir pruebas automatizadas sólidas y de alta señal. Úselo cuando pregunte "¿Cómo pruebo esto de manera efectiva?", arregle pruebas inestables, refactorice conjuntos de pruebas o decida entre estrategias de unidad/integración/E2E. distinto de TDD (proceso); Esta habilidad se centra en la calidad y la arquitectura de las pruebas.

Effectuez des audits de sécurité sur les modifications de code, les différences ou les branches pour trouver des vulnérabilités exploitables en toute confiance. À utiliser lorsqu'on vous demande de « auditer la sécurité », « examiner les vulnérabilités », « analyser la sécurité », « vérifier les problèmes de sécurité », « auditer ce PR », « examiner ces modifications pour la sécurité » ou « rechercher des vulnérabilités dans les différences ». À la différence de l'examen de sécurité (qui fournit des modèles/listes de contrôle de codage sécurisés) : cette compétence audite activement les modifications de code à l'aide d'une méthodologie structurée avec filtrage des faux positifs. Inclut des scripts Python pour l'intégration de GitHub Action CI et l'évaluation des relations publiques.

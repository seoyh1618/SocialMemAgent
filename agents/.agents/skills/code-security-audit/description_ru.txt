Выполняйте аудит безопасности изменений кода, различий или ветвей, чтобы найти уязвимости, которые можно использовать с высокой степенью достоверности. Используйте, когда вас просят «проверить безопасность», «проверить на наличие уязвимостей», «сканировать безопасность», «проверить наличие проблем с безопасностью», «проверить этот PR», «просмотреть эти изменения на предмет безопасности» или «найти уязвимости в diff». В отличие от проверки безопасности (которая предоставляет безопасные шаблоны/контрольные списки кодирования) — этот навык активно проверяет изменения кода, используя структурированную методологию с фильтрацией ложных срабатываний. Включает скрипты Python для интеграции GitHub Action CI и оценки связей с общественностью.

Chargez de manière PROACTIVE lorsque la tâche implique d'optimiser la vitesse, de réduire la taille du paquet ou d'améliorer la réactivité. À utiliser lorsque l'utilisateur dit « accélérer », « réduire la taille du bundle », « corriger les requêtes lentes », « optimiser le rendu » ou « vérifier les Core Web Vitals ». Couvre l'analyse des bundles et l'arborescence, l'optimisation des requêtes de base de données (N+1, indexation), les performances de rendu React (rendu, mémorisation), l'optimisation de la cascade de réseau, la détection des fuites de mémoire, les performances côté serveur et l'amélioration de Core Web Vitals (LCP, FID, CLS).

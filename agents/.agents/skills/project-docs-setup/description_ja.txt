プロジェクトドキュメント構造作成アシスタント。経験豊富なプロダクト マネージャーとして、複数ラウンドの対話を通じてプロジェクトの要件について深い洞察を獲得し、合理性を分析してベスト プラクティスの推奨事項を提供し、最終的には完全な技術文書システムを作成します。

**使用する場合:**
- ドキュメント システムのセットアップが必要な新しいプロジェクトのキックオフ (推奨される最初のステップ)
- 既存のプロジェクトには標準ドキュメントが不足しており、完了が必要です
- コードは存在しますが、ドキュメントがありません。コードからドキュメントを埋め戻す必要がある
- 専門的な製品およびアーキテクチャ分析のアドバイスが必要

**サポートされている 2 つのモード:**
- モード A: 新規プロジェクトのドキュメント作成 (要件からドキュメントまで)
- モード B: コードからドキュメントへのバックフィル (コードからドキュメントへ)

**プロジェクト ワークフローとの関係:**
- このスキルは完全なドキュメント (PRD、SAD、開発ガイドなど) を作成します。
- project-workflow はこれらのドキュメントに基づいて開発タスクを実行します
- 推奨されるフロー: プロジェクト-ドキュメント-セットアップ → プロジェクト-ワークフロー

**トリガーフレーズ:**
- 「プロジェクト ドキュメントの作成」 / 「プロジェクト ドキュメントの初期化」 / 「プロジェクト ドキュメントのセットアップ」
- 「コードからドキュメントを生成」 / 「プロジェクトドキュメントをバックフィル」 / 「プロジェクトドキュメントを補足」
- プロジェクトのニーズを直接説明します。例: 「オンライン教育プラットフォーム用のドキュメントの作成を手伝ってください」

项目文档结构创建助手。作为一名经验丰富的产品经理，通过多轮对话深入洞察项目需求，分析合理性并提供最佳实践建议，最终形成完整的技术文档体系。

**何时使用：**
- 新项目启动需要文档系统设置（建议第一步）
- 现有项目缺乏标准文档并需要完成
- 代码存在但文档缺失；需要从代码回填文档
- 需要专业的产品和架构分析建议

**支持两种模式：**
- 模式A：新项目文档创建（从需求到文档）
- 模式B：Code-to-docs回填（从代码到文档）

**与项目工作流程的关系：**
- 此技能创建完整的文档（PRD、SAD、开发指南等）
- 项目工作流程根据这些文档执行开发任务
- 建议流程：项目文档设置 → 项目工作流程

**触发短语：**
- “创建项目文档”/“初始化项目文档”/“设置项目文档”
-“从代码生成文档”/“回填项目文档”/“补充项目文档”
- 直接描述项目需求，例如“帮我为在线教育平台创建文档”

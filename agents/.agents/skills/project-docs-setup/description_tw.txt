專案文檔結構建立助理。作為一名經驗豐富的產品經理，透過多輪對話深入洞察專案需求，分析合理性並提供最佳實踐建議，最終形成完整的技術文件系統。

**何時使用：**
- 新專案啟動需要文件系統設定（建議第一步）
- 現有項目缺乏標準文件並需要完成
- 程式碼存在但文檔缺失；需從程式碼回填文檔
- 需要專業的產品和架構分析建議

**支援兩種模式：**
- 模式A：新專案文件建立（從需求到文件）
- 模式B：Code-to-docs回填（從程式碼到文件）

**與專案工作流程的關係：**
- 此技能可建立完整的文件（PRD、SAD、開髮指南等）
- 專案工作流程根據這些文件執行開發任務
- 建議流程：專案文件設定 → 專案工作流程

**觸發短語：**
- “建立專案文件”/“初始化專案文件”/“設定專案文件”
-“從程式碼產生文件”/“回填專案文件”/“補充專案文件”
- 直接描述專案需求，例如“幫我為線上教育平台建立文件”

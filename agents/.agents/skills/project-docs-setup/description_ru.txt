Помощник по созданию структуры проектной документации. Будучи опытным менеджером по продукту, вы сможете получить глубокое представление о требованиях проекта посредством многоэтапного диалога, проанализировать обоснованность и предоставить рекомендации по передовому опыту, в конечном итоге создав полную систему технической документации.

**Когда использовать:**
- Начало нового проекта, требующее настройки системы документации (рекомендуемый первый шаг)
- Существующему проекту не хватает стандартной документации и он требует завершения.
- Код существует, но документация отсутствует; нужно заполнить документы из кода
- Нужна профессиональная консультация по анализу продукта и архитектуры.

**Два поддерживаемых режима:**
- Режим А: Создание документации нового проекта (от требований до документов)
- Режим B: обратная загрузка кода в документы (из кода в документы).

**Связь с рабочим процессом проекта:**
- Этот навык позволяет создавать полную документацию (PRD, SAD, руководства для разработчиков и т. д.).
- рабочий процесс проекта выполняет задачи разработки на основе этих документов.
– Рекомендуемый порядок действий: настройка проектной документации → рабочий процесс проекта.

**Триггерные фразы:**
- «Создать документацию проекта» / «Инициализировать документацию проекта» / «Настроить документацию проекта»
- «Создать документацию из кода» / «Заполнить документацию проекта» / «Дополнить документацию проекта»
- Непосредственно опишите потребности проекта, например. «Помогите мне создать документы для платформы онлайн-образования»

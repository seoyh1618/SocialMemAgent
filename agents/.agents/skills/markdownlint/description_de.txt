Gewährleistet Stilkonsistenz und Korrektheit in Markdown-Dateien mithilfe von markdownlint-cli2. Verwenden Sie diese Option immer dann, wenn Sie eine Markdown-Datei (.md) erstellen, aktualisieren, bearbeiten oder überprüfen. Wird bei jeder Aufgabe ausgelöst, die .md-Dateien erstellt oder ändert, einschließlich README.md, CHANGELOG.md, Dokumentation oder jeder Datei mit der Erweiterung .md. Wird auch ausgelöst, wenn der Benutzer Markdown, Linting, Markdown-Formatierung, Markdown-Stil oder Markdownlint erwähnt. Führen Sie den Linter immer aus, nachdem Sie Markdown-Inhalte geschrieben oder bearbeitet haben – überspringen Sie diesen Schritt nicht.

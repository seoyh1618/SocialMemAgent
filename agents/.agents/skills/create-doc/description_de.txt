Erstellen Sie Dokumentationen gemäß den Docs-as-Code-Konventionen. TRIGGER, wenn: Der Benutzer möchte Dokumentation, Diagramme oder technische Dokumente erstellen, schreiben oder entwerfen. Beispiele: „Erstelle ein Dokument für …“, „Architekturdokumentation schreiben“, „Füge ein C4-Diagramm hinzu“, „Dokumentiere das Domänenmodell“. NICHT AUSLÖSEN, wenn: Der Benutzer möchte eine ADR (verwenden Sie „create-adr“), eine Spezifikation (verwenden Sie „create-spec“) oder möchte einfach nur vorhandene Dokumente lesen.

Implementierung + Prüfschleife unter Verwendung paralleler Agententeams mit strukturierten Vereinfachungs-, Härtungs- und Dokumentdurchgängen. Erzeugt Implementierungsagenten, die die Arbeit erledigen, dann Prüfagenten, um Komplexität, Sicherheitslücken und Spezifikationsabweichungen zu finden, und führt dann Schleifen durch, bis der Code sauber kompiliert wird, alle Tests bestanden werden und Prüfer keine Probleme finden oder die Schleifenobergrenze erreicht ist. Verwenden Sie diese Option, wenn Sie Funktionen aus einer Spezifikation oder einem Plan implementieren, vorhandenen Code härten, eine Reihe von Problemen beheben oder eine Aufgabe mit mehreren Dateien ausführen möchten, die von einem Build-Überprüfung-Fix-Zyklus profitiert.

Una herramienta automatizada para generar por lotes y mantener la documentación de la biblioteca de componentes del frontend. Diseñado para bibliotecas de componentes de Monorepo (por ejemplo, atom-ui-mobile), compara los tiempos de modificación de los archivos de componentes y sus documentos para generar automáticamente listas de tareas y producir por lotes documentación completa de componentes que cumpla con los estándares. Admite: (1) actualizaciones periódicas por lotes de documentos de componentes, (2) mantenimiento de documentos de un solo componente, (3) inicialización de documentos de nuevos componentes. Identifica automáticamente los archivos del componente principal, omitiendo archivos auxiliares como index.ts, store, hooks, etc.; Los documentos se encuentran en index.zh-CN.md dentro del directorio de cada componente.

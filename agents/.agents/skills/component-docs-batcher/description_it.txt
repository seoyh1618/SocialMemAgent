Uno strumento automatizzato per la generazione batch e il mantenimento della documentazione della libreria dei componenti frontend. Progettato per le librerie di componenti Monorepo (ad esempio atom-ui-mobile), confronta i tempi di modifica dei file dei componenti e dei relativi documenti per generare automaticamente elenchi di attività e produrre in batch la documentazione completa dei componenti conforme agli standard. Supporta: (1) aggiornamenti batch periodici dei documenti dei componenti, (2) manutenzione dei documenti a componente singolo, (3) inizializzazione dei documenti dei nuovi componenti. Identifica automaticamente i file dei componenti principali, saltando i file ausiliari come index.ts, store, hook, ecc.; i documenti vengono inseriti in index.zh-CN.md all'interno della directory di ciascun componente.

Leitfaden zur Implementierung der Service Worker API – Registrierung, Lebenszyklusverwaltung, Caching-Strategien, Push-Benachrichtigungen und Hintergrundsynchronisierung. Wird verwendet, wenn: (1) Service-Worker-Dateien (sw.js) erstellt oder geändert werden, (2) Offline-First-Caching implementiert wird (Cache-First, Network-First, Stale-While-Revalidate), (3) Push-Benachrichtigungen oder Hintergrundsynchronisierung eingerichtet werden, (4) Service-Worker-Registrierung, -Bereich oder Update-Probleme behoben werden, (5) Navigations-Preload implementiert wird, (6) Benutzer erwähnt „Service Worker“, „sw.js“, „Offline-Support“, „Cache-Strategie“, „Push-Benachrichtigung“, „Hintergrundsynchronisierung“, „Workbox-Alternative“ oder „PWA-Caching“.

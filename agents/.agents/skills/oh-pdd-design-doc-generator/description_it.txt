Prima della generazione, analizza la base di codice OpenHarmony esistente per garantire la compatibilità con l'architettura attuale. Applicabile alle richieste degli utenti: (1) generare un documento di progettazione dell'architettura, (2) generare un documento di progettazione funzionale, (3) generare un documento di progettazione da PRD, (4) creare una progettazione dell'architettura di sistema, (5) scrivere specifiche funzionali, (6) analizzare la codebase OH.

Générez automatiquement des documents de conception HarmonyOS, y compris des documents de conception architecturale et de conception fonctionnelle, basés sur les documents PRD. Avant la génération, il analyse la base de code OpenHarmony existante pour garantir la compatibilité avec l'architecture actuelle. Le chapitre 2 du document de conception d'architecture doit être une analyse de la solution concurrente et doit apparaître après le contexte des exigences. Applicable aux demandes des utilisateurs : (1) générer un document de conception d'architecture, (2) générer un document de conception fonctionnelle, (3) générer un document de conception à partir du PRD, (4) créer une conception d'architecture système, (5) écrire une spécification fonctionnelle, (6) analyser la base de code OH. Mots-clés : conception d'architecture, conception fonctionnelle, document de conception, analyse des concurrents, analyse du code OpenHarmony, conception d'architecture, conception fonctionnelle, génération de documents de conception, analyse du code OH, analyse de la base de code, analyse des concurrents

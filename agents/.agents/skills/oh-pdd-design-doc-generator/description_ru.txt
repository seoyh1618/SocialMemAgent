Автоматически создавайте проектные документы HarmonyOS, включая документы по архитектуре и функциональному проектированию, на основе документов PRD. Перед генерацией он анализирует существующую кодовую базу OpenHarmony, чтобы обеспечить совместимость с текущей архитектурой. Глава 2 документа по проектированию архитектуры должна представлять собой анализ решений конкурентов и располагаться после описания требований. Применимо к запросам пользователей: (1) создать документ о проектировании архитектуры, (2) создать документ о функциональном проектировании, (3) создать документ о проектировании из PRD, (4) создать проект архитектуры системы, (5) написать функциональную спецификацию, (6) проанализировать кодовую базу OH. Ключевые слова: проектирование архитектуры, функциональное проектирование, проектная документация, анализ конкурентов, анализ кода OpenHarmony, проектирование архитектуры, функциональное проектирование, генерация проектной документации, анализ кода OH, анализ кодовой базы, анализ конкурентов.

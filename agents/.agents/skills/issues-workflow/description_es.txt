Planifique y realice un seguimiento del trabajo utilizando un flujo de trabajo de GitHub Issues-first con jerarquías de sub-temas. Úselo siempre que el usuario necesite organizar problemas en jerarquías padre-hijo, administrar subproblemas, realizar un seguimiento de la investigación en varios problemas o coordinar flujos de trabajo de problemas entre repositorios. Úselo también cuando el usuario pregunte sobre la estrategia de seguimiento de problemas o cómo estructurar el desglose del trabajo en GitHub. NO lo use para crear problemas individuales (use en su lugar crear problemas) o para la administración del tablero de Proyectos de GitHub.

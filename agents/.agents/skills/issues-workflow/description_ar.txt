قم بتخطيط وتتبع العمل باستخدام سير عمل GitHub Issues-first مع التسلسلات الهرمية للمشكلات الفرعية. يُستخدم عندما يحتاج المستخدم إلى تنظيم المشكلات في تسلسلات هرمية بين الوالدين والطفل، أو إدارة المشكلات الفرعية، أو تتبع البحث عبر مشكلات متعددة، أو تنسيق سير عمل مشكلات الريبو المتبادل. يُستخدم أيضًا عندما يسأل المستخدم عن استراتيجية تتبع المشكلات أو كيفية تنظيم تقسيم العمل في GitHub. لا تستخدم لإنشاء مشكلات فردية (استخدم إنشاء المشكلات بدلاً من ذلك) أو لإدارة مجلس إدارة مشاريع GitHub.

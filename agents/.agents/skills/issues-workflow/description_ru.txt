Планируйте и отслеживайте работу, используя рабочий процесс GitHub «Сначала проблемы» с иерархией подзадач. Используйте его всякий раз, когда пользователю необходимо организовать проблемы в иерархии «родители-потомки», управлять подзадачами, отслеживать исследования по нескольким проблемам или координировать рабочие процессы по проблемам перекрестного репозитория. Также используйте, когда пользователь спрашивает о стратегии отслеживания проблем или о том, как структурировать работу в GitHub. НЕ используйте для создания отдельных задач (вместо этого используйте Issue-create) или для управления досками проектов GitHub.

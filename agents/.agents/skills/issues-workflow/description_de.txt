Planen und verfolgen Sie die Arbeit mit einem GitHub Issues-First-Workflow mit Unter-Issue-Hierarchien. Verwenden Sie es immer dann, wenn der Benutzer Vorgänge in übergeordneten-untergeordneten Hierarchien organisieren, Untervorgänge verwalten, Recherchen über mehrere Vorgänge hinweg verfolgen oder Repository-übergreifende Vorgangsabläufe koordinieren muss. Verwenden Sie es auch, wenn der Benutzer nach einer Strategie zur Problemverfolgung oder zur Strukturierung der Arbeitsaufteilung in GitHub fragt. NICHT zum Erstellen einzelner Issues (verwenden Sie stattdessen Issue-Create) oder für die Board-Verwaltung von GitHub-Projekten verwenden.

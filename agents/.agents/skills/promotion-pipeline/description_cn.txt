OCI 工件升级管道，从 PR 合并到集成验证再到实时部署。

在以下情况下使用：(1) 跟踪更改尚未到达集群的原因，(2) 调试集成中卡住的工件，
(3) 了解构建/验证/升级生命周期，(4) 执行手动升级或回滚，
(5) 调查 GitHub Actions 工作流程故障，(6) 检查 GHCR 中的 OCI 工件标签。

OCI-Artefakt-Promotion-Pipeline von der PR-Zusammenführung über die Integrationsvalidierung bis hin zur Live-Bereitstellung.

Wird verwendet, wenn: (1) nachverfolgt werden soll, warum eine Änderung einen Cluster nicht erreicht hat, (2) Artefakte debuggen, die in der Integration hängen geblieben sind,
(3) Den Build-/Validierungs-/Hochstufungslebenszyklus verstehen, (4) Manuelle Hochstufung oder Rollback durchführen,
(5) Untersuchen von GitHub Actions-Workflow-Fehlern, (6) Überprüfen von OCI-Artefakt-Tags in GHCR.

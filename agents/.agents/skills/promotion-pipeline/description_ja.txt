PR マージから統合検証を経てライブ デプロイメントまでの OCI アーティファクト プロモーション パイプライン。

次の場合に使用します: (1) 変更がクラスターに到達しなかった理由を追跡する、(2) 統合でスタックしたアーティファクトをデバッグする、
(3) ビルド/検証/プロモートのライフサイクルを理解する、(4) 手動でプロモートまたはロールバックを実行する、
(5) GitHub Actions ワークフローの失敗を調査する、(6) GHCR で OCI アーティファクト タグを確認する。

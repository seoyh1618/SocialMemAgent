OCI 工件升級管道，從 PR 合併到整合驗證再到即時部署。

在下列情況下使用：(1) 追蹤變更尚未到達叢集的原因，(2) 調試整合中卡住的工件，
(3) 了解建置/驗證/升級生命週期，(4) 執行手動升級或回滾，
(5) 調查 GitHub Actions 工作流程故障，(6) 檢查 GHCR 中的 OCI 工件標籤。

Guide pour créer, configurer et déployer des microfrontends sur Vercel. Utilisez cette compétence lorsque l'utilisateur mentionne les microfrontends, les multizones, la division d'une application en équipes, les déploiements indépendants, le routage inter-applications, la migration incrémentielle, la composition de plusieurs frontends sous un seul domaine, microfrontends.json, @vercel/microfrontends, le proxy local des microfrontends ou le routage basé sur le chemin entre les projets Vercel. À utiliser également lorsque l'utilisateur pose des questions sur les mises en page partagées entre les projets, la navigation entre les microfrontends, les environnements de secours, les préfixes d'actifs ou le routage contrôlé par indicateur de fonctionnalité.

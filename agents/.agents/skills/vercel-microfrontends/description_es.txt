Guía para construir, configurar e implementar microfrontends en Vercel. Utilice esta habilidad cuando el usuario mencione microfrontends, zonas múltiples, dividir una aplicación entre equipos, implementaciones independientes, enrutamiento entre aplicaciones, migración incremental, composición de múltiples frontends bajo un dominio, microfrontends.json, @vercel/microfrontends, el proxy local de microfrontends o enrutamiento basado en rutas entre proyectos de Vercel. Utilícelo también cuando el usuario pregunte sobre diseños compartidos entre proyectos, navegación entre microfrontends, entornos alternativos, prefijos de activos o enrutamiento controlado por indicadores de funciones.

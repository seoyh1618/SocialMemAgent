Leitfaden zum Erstellen, Konfigurieren und Bereitstellen von Mikrofrontends auf Vercel. Verwenden Sie diese Fähigkeit, wenn der Benutzer Microfrontends, Multizonen, die Aufteilung einer App auf mehrere Teams, unabhängige Bereitstellungen, Cross-App-Routing, inkrementelle Migration, das Zusammenstellen mehrerer Frontends unter einer Domäne, microfrontends.json, @vercel/microfrontends, den lokalen Proxy des Microfrontends oder pfadbasiertes Routing zwischen Vercel-Projekten erwähnt. Verwenden Sie es auch, wenn der Benutzer nach projektübergreifenden gemeinsamen Layouts, Navigation zwischen Microfrontends, Fallback-Umgebungen, Asset-Präfixen oder Feature-Flag-gesteuertem Routing fragt.

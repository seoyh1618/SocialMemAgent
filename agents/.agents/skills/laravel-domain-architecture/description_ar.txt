بنية موجهة نحو المجال لـ Laravel بدون تبعيات الحزمة الخارجية، استنادًا إلى مفاهيم من "Laravel Beyond CRUD" (Spatie). استخدم هذه المهارة عندما يريد المستخدم بناء مشروع Laravel باستخدام DDD، أو البنية الموجهة للمجال، أو تنظيم التعليمات البرمجية حسب مفاهيم الأعمال بدلاً من الخصائص التقنية. يتم التنشيط أيضًا عندما يسأل المستخدم عن: الإجراءات، أو كائنات البيانات (DTOs)، أو النماذج البسيطة، أو الحالات ذات التعدادات، أو ViewModels (Blade أو API)، أو منشئي استعلام HTTP، أو طبقة التطبيق، أو توسيع نطاق مشاريع Laravel بما يتجاوز CRUD البسيط. قم بالتنشيط لأي ذكر لـ "طبقة المجال"، أو "طبقة التطبيق"، أو "ما وراء CRUD"، أو "الموجه نحو المجال"، أو طلبات إعادة هيكلة مشروع Laravel في بنية المجال. حتى لو سأل المستخدم "كيفية تنظيم مشروع Laravel كبير"، فإن هذه المهارة ذات صلة.

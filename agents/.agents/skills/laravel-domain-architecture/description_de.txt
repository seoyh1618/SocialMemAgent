Domänenorientierte Architektur für Laravel ohne externe Paketabhängigkeiten, basierend auf Konzepten von „Laravel Beyond CRUD“ (Spatie). Verwenden Sie diese Fähigkeit, wenn der Benutzer ein Laravel-Projekt mithilfe von DDD, einer domänenorientierten Architektur, strukturieren oder Code nach Geschäftskonzepten statt nach technischen Eigenschaften organisieren möchte. Aktivieren Sie die Funktion auch, wenn der Benutzer Folgendes fragt: Aktionen, Datenobjekte (DTOs), Lean-Modelle, Zustände mit Aufzählungen, ViewModels (Blade oder API), HTTP-Abfrage-Builder, Anwendungsschicht oder die Skalierung von Laravel-Projekten über einfaches CRUD hinaus. Aktivieren Sie diese Option bei jeder Erwähnung von „Domänenschicht“, „Anwendungsschicht“, „über CRUD hinaus“, „domänenorientiert“ oder bei Anfragen zur Umgestaltung eines Laravel-Projekts in eine Domänenstruktur. Auch wenn der Benutzer fragt, „wie man ein großes Laravel-Projekt organisiert“, ist diese Fähigkeit relevant.

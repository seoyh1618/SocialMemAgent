Architecture orientée domaine pour Laravel sans dépendances de packages externes, basée sur les concepts de "Laravel Beyond CRUD" (Spatie). Utilisez cette compétence lorsque l'utilisateur souhaite structurer un projet Laravel à l'aide de DDD, d'une architecture orientée domaine ou organiser le code par concepts métier plutôt que par propriétés techniques. Activez également lorsque l'utilisateur pose des questions sur : les actions, les objets de données (DTO), les modèles allégés, les états avec des énumérations, les ViewModels (Blade ou API), les générateurs de requêtes HTTP, la couche d'application ou la mise à l'échelle des projets Laravel au-delà du simple CRUD. Activez pour toute mention de « couche de domaine », « couche d'application », « au-delà de CRUD », « orienté domaine » ou demandes de refactorisation d'un projet Laravel en structure de domaine. Même si l'utilisateur demande « comment organiser un grand projet Laravel », cette compétence est pertinente.

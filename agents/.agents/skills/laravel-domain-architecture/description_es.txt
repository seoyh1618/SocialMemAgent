Arquitectura orientada a dominio para Laravel sin dependencias de paquetes externos, basada en conceptos de "Laravel Beyond CRUD" (Spatie). Utilice esta habilidad cuando el usuario desee estructurar un proyecto de Laravel utilizando DDD, arquitectura orientada a dominios u organizar el código por conceptos comerciales en lugar de propiedades técnicas. También se activa cuando el usuario pregunta sobre: ​​Acciones, Objetos de datos (DTO), modelos lean, Estados con enumeraciones, ViewModels (Blade o API), Constructores de consultas HTTP, capa de aplicación o escalado de proyectos Laravel más allá del simple CRUD. Actívelo para cualquier mención de "capa de dominio", "capa de aplicación", "más allá de CRUD", "orientado a dominio" o solicitudes para refactorizar un proyecto de Laravel en una estructura de dominio. Incluso si el usuario pregunta "cómo organizar un gran proyecto Laravel", esta habilidad es relevante.

「Laravel Beyond CRUD」(Spatie) の概念に基づいた、外部パッケージの依存関係のない Laravel のドメイン指向アーキテクチャ。ユーザーが DDD、ドメイン指向アーキテクチャを使用して Laravel プロジェクトを構築したい場合、または技術的特性ではなくビジネス概念別にコードを整理したい場合に、このスキルを使用します。また、アクション、データ オブジェクト (DTO)、リーン モデル、列挙型の状態、ViewModel (ブレードまたは API)、HTTP クエリ ビルダー、アプリケーション層、または単純な CRUD を超えた Laravel プロジェクトのスケーリングについてユーザーが尋ねたときにもアクティブ化します。 「ドメイン層」、「アプリケーション層」、「CRUD を超えて」、「ドメイン指向」について言及する場合、または Laravel プロジェクトをドメイン構造にリファクタリングする要求がある場合にアクティブ化します。ユーザーが「大規模な Laravel プロジェクトを編成する方法」を尋ねた場合でも、このスキルは役に立ちます。

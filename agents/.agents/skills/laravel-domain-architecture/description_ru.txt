Доменно-ориентированная архитектура для Laravel без зависимостей от внешних пакетов, основанная на концепциях «Laravel Beyond CRUD» (Spatie). Используйте этот навык, когда пользователь хочет структурировать проект Laravel с использованием DDD, предметно-ориентированной архитектуры или организовать код по бизнес-концепциям, а не по техническим свойствам. Также активируйте, когда пользователь спрашивает о: действиях, объектах данных (DTO), экономичных моделях, состояниях с перечислениями, моделях представления (Blade или API), конструкторах HTTP-запросов, уровне приложений или масштабировании проектов Laravel за пределы простого CRUD. Активируйте при любом упоминании «уровня домена», «уровня приложения», «за пределами CRUD», «ориентированного на предметную область» или запросов на рефакторинг проекта Laravel в структуру домена. Даже если пользователь спрашивает «как организовать большой проект в Laravel», этот навык актуален.

Architettura orientata al dominio per Laravel senza dipendenze di pacchetti esterni, basata sui concetti di "Laravel Beyond CRUD" (Spatie). Utilizza questa competenza quando l'utente desidera strutturare un progetto Laravel utilizzando DDD, architettura orientata al dominio o organizzare il codice in base a concetti aziendali anziché a proprietà tecniche. Si attiva anche quando l'utente chiede informazioni su: azioni, oggetti dati (DTO), modelli snelli, stati con enumerazioni, ViewModel (Blade o API), builder di query HTTP, livello di applicazione o ridimensionamento di progetti Laravel oltre il semplice CRUD. Attivare per qualsiasi menzione di "livello di dominio", "livello di applicazione", "oltre CRUD", "orientato al dominio" o richieste di refactoring di un progetto Laravel nella struttura del dominio. Anche se l'utente chiede "come organizzare un grande progetto Laravel", questa abilità è rilevante.

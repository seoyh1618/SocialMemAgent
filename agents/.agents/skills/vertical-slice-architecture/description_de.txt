Erzwingen Sie die Vertical Slice Architecture (VSA) beim Erstellen von Anwendungen in jeder Sprache (Go, .NET/C#, Java, Kotlin, TypeScript, Python usw.) und jedem Typ (Web-API, mobiles Backend, CLI, ereignisgesteuert). Organisieren Sie Code nach Funktion/Anwendungsfall statt nach technischen Ebenen. Jedes Feature ist ein eigenständiges vertikales Segment mit einem einzelnen Einstiegspunkt, der das Router-/Framework-Handle und seine Abhängigkeiten empfängt. Verwenden Sie diese Option, wenn der Benutzer „vertikale Slice-Architektur“, „VSA“, „für Feature organisieren“, „Feature-basierte Architektur“, „Slice-Architektur“ sagt oder wenn Sie eine neue App oder Funktion erstellen und das Projekt bereits VSA-Konventionen befolgt. Verwenden Sie es auch, wenn Sie Code überprüfen oder umgestalten, um ihn an die VSA-Prinzipien anzupassen.

Appliquez l'architecture de tranche verticale (VSA) lors de la création d'applications dans n'importe quel langage (Go, .NET/C#, Java, Kotlin, TypeScript, Python, etc.) et tout type (API Web, backend mobile, CLI, piloté par événements). Organisez le code par fonctionnalité/cas d'utilisation plutôt que par couches techniques. Chaque fonctionnalité est une tranche verticale autonome avec un point d’entrée unique qui reçoit le handle du routeur/framework et ses dépendances. À utiliser lorsque l'utilisateur dit « architecture de tranche verticale », « VSA », « organiser une fonctionnalité », « architecture basée sur les fonctionnalités », « architecture de tranche » ou lors de la création d'une nouvelle application ou fonctionnalité et que le projet suit déjà les conventions VSA. À utiliser également lors de la révision ou de la refactorisation du code pour l'aligner sur les principes VSA.

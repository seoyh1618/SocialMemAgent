Поддержка проектирования и проверки проектов, использующих чистую архитектуру. Основан на четырехуровневой структуре (уровень домена, уровень вариантов использования, уровень адаптера интерфейса, уровень инфраструктуры). В частности, уровень инфраструктуры должен содержать только сквозные задачи (ведение журнала, управление конфигурацией), тогда как сохранение и RPC должны быть помещены на уровень адаптера интерфейса. Триггер: активируется только при явном упоминании «Чистой архитектуры», например «с чистой архитектурой», «в соответствии с чистой архитектурой» или «обзор чистой архитектуры». Он не активируется для общих «обзоров дизайна» или «консультаций по архитектуре».

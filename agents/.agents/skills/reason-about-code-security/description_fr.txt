Développer un raisonnement systématique sur les menaces et une réflexion contradictoire sur la sécurité du code. À utiliser lorsqu'un apprenant souhaite analyser le code pour en déterminer les implications en matière de sécurité, comprendre les modèles de vulnérabilité ou développer une réflexion axée sur la sécurité. Cette compétence enseigne la modélisation systématique des menaces, la formulation d'hypothèses et le raisonnement de défense, et non le catalogage des vulnérabilités. Se déclenche sur des expressions telles que « est-ce sécurisé », « implications en matière de sécurité », « cela pourrait-il être exploité », « analyse des menaces » ou lorsqu'un apprenant souhaite développer des compétences de raisonnement en matière de sécurité.

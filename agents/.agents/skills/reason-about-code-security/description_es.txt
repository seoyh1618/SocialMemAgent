Desarrollar un razonamiento sistemático sobre amenazas y un pensamiento contradictorio sobre la seguridad del código. Úselo cuando un alumno quiera analizar el código en busca de implicaciones de seguridad, comprender patrones de vulnerabilidad o desarrollar un pensamiento centrado en la seguridad. Esta habilidad enseña el modelado sistemático de amenazas, la aparición de suposiciones y el razonamiento de defensa, no la catalogación de vulnerabilidades. Se activan frases como "¿es esto seguro?", "implicaciones para la seguridad", "podría explotarse", "análisis de amenazas" o cuando un alumno desea desarrollar habilidades de razonamiento de seguridad.

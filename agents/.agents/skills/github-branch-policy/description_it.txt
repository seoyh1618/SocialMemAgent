Controlla la governance del ramo del repository GitHub e l'igiene del flusso di lavoro. Da utilizzare quando viene richiesto di rivedere set di regole, controlli di stato richiesti, restrizioni di aggiornamento, comportamento di bypass/aggiornamento, impostazioni di eliminazione al momento dell'unione, affidabilità del flusso di lavoro di unione automatica, rami obsoleti, registrazioni di flussi di lavoro fantasma o deviazione delle policy dei rami.

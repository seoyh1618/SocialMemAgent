قم بمسح وتحليل مستودع البرامج أو المشروع للتأكد من جودة التصميم باستخدام مبادئ من فلسفة تصميم البرمجيات لجون أوسترهوت. يُستخدم عندما يطلب المستخدم مراجعة أو تدقيق أو مسح أو تقييم جودة التعليمات البرمجية أو جودة التصميم أو البنية أو الدين الفني. قم أيضًا بتشغيل ما يلي: مراجعة التعليمات البرمجية، ومراجعة التصميم، وتحليل التعقيد، والتحقق من صحة التعليمات البرمجية، وتحليل عمق الوحدة، ومراجعة إخفاء المعلومات، ومدى جودة التعليمات البرمجية الخاصة بي، ومراجعة مشروعي، والعثور على مشاكل التصميم، وما هو الخطأ في قاعدة التعليمات البرمجية الخاصة بي، وتقييم التعليمات البرمجية الخاصة بي، أو أي شيء يتعلق بتقييم جودة تصميم البرامج على المستوى الهيكلي. هذا ليس مدققًا أو مدققًا للأسلوب. يقوم بتقييم صفات التصميم العميقة مثل عمق الوحدة وجودة التجريد وإخفاء المعلومات وأنماط التعقيد.

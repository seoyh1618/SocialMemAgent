John Ousterhout 著の「A Philosophy of Software Design」の原則を使用して、ソフトウェア リポジトリまたはプロジェクトをスキャンして設計品質を分析します。ユーザーがコード品質、設計品質、アーキテクチャ、または技術的負債のレビュー、監査、スキャン、または評価を要求する場合に使用します。コード レビュー、設計レビュー、複雑さの分析、コードの健全性チェック、モジュールの深さの分析、情報隠蔽のレビュー、コードの良し悪し、プロジェクトのレビュー、設計の問題の発見、コードベースの問題点、コードの評価、または構造レベルでのソフトウェア設計の品質の評価に関するあらゆる目的にもトリガーされます。これはリンターやスタイル チェッカーではありません。モジュールの深さ、抽象化の品質、情報の隠蔽、複雑さのパターンなどの詳細な設計品質を評価します。

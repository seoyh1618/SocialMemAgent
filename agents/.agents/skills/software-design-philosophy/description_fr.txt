Gérez la complexité des logiciels grâce à des modules approfondis, à la dissimulation d'informations et à une programmation stratégique. À utiliser lorsque l'utilisateur mentionne « conception de module », « API trop complexe », « classe peu profonde », « budget de complexité » ou « stratégique ou tactique ». Couvre les modules profonds et peu profonds, les signaux d'alarme en cas de complexité et les commentaires en tant que documentation de conception. Pour la qualité du code, voir clean-code. Pour les limites, voir architecture propre.

Estilo de código, patrones de arquitectura y guía de desarrollo para el backend de comercio electrónico Droplinked.
NestJS + Prisma (MongoDB) con arquitectura en capas estricta: Controlador a Fachada de Servicio, Caso de Uso y Repositorio.

Utilice esta habilidad cuando:
(1) Crear nuevas funciones, módulos o puntos finales en el backend de Droplinked
(2) Refactorizar el código existente para seguir los patrones del proyecto
(3) Redacción de pruebas para casos de uso y servicios.
(4) Revisión del código para el cumplimiento arquitectónico
(5) Comprender el modelo de datos (tiendas, productos, pedidos, carritos, comerciantes, clientes)
(6) Implementación de comunicación entre módulos
(7) Agregar efectos secundarios basados en eventos
(8) Trabajar con Prisma y MongoDB en este código base

Activadores: droplinked, módulo nido, caso de uso, fachada de servicio, patrón de repositorio, carrito, pedido, tienda, producto, comerciante

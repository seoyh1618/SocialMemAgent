Convenzioni di denominazione dell'SDK OpenLark Rust e di espressione API esterna (client/servizio/risorsa/richiesta/costruttore). Utilizzato per aggiungere/refactoring di tipi pubblici, progettare catene di meta chiamate, regolare le esportazioni e il preludio dei moduli o indagare su problemi come la confusione semantica/nome DocsService e gli stili di chiamata incoerenti in openlark-docs. Parole chiave di attivazione: convenzioni di denominazione, client vs servizio, risorsa, ridenominazione, catena di meta chiamate, API pubblica

Convenciones de expresión de API pública y nomenclatura del SDK de OpenLark Rust (Cliente/Servicio/Recurso/Solicitud/Constructor). Se utiliza para agregar/refactorizar tipos públicos, diseñar cadenas de metallamadas, ajustar las exportaciones y el preludio de módulos, o investigar problemas como conflictos de nombres de DocsService/confusión semántica y estilos de llamadas inconsistentes en openlark-docs. Palabras clave de activación: convenciones de nomenclatura, cliente frente a servicio, recurso, cambio de nombre, cadena de metallamadas, API pública

Стандарты разработки на основе документов требуют, чтобы агент ознакомился с официальной документацией и примерами перед созданием кода или исправлением ошибок, включая процедуры проверки API, стратегии поиска и правила вызова MCP. Применимо к таким сценариям, как интеграция сторонних библиотек, устранение ошибок API и обработка изменений версий.

Dokumentgesteuerte Entwicklungsrichtlinien erfordern, dass der Agent die offizielle Dokumentation und Beispiele konsultiert, bevor er Code generiert oder Fehler behebt, einschließlich API-Verifizierungsverfahren, Suchstrategien und MCP-Aufrufregeln. Anwendbar auf Szenarien wie die Integration von Bibliotheken von Drittanbietern, die Fehlerbehebung bei API-Fehlern und die Handhabung von Versionsänderungen.

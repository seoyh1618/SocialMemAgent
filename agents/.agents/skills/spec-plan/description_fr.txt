Lorsque l'utilisateur souhaite effectuer un développement basé sur les spécifications, vous DEVEZ utiliser cette compétence : faites d'abord converger les exigences et les contraintes dans une spécification implémentable et révisable, puis procédez à la mise en œuvre. Cette compétence produit un seul fichier de spécifications écrit dans /docs/specs/ et effectue un seul commit git (en validant uniquement cette spécification). Cette compétence n'écrit aucun code d'implémentation.

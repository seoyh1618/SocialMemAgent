Cuando el usuario quiera realizar un desarrollo basado en especificaciones, DEBE utilizar esta habilidad: primero converger los requisitos y restricciones en una especificación implementable y revisable, luego proceder a la implementación. Esta habilidad produce solo un archivo de especificaciones escrito en /docs/specs/ y realiza una confirmación de git (confirmando solo esa especificación). Esta habilidad no escribe código de implementación.
